### TC-003: 关键 smoke 用例顺序回归通过
- **结果**: FAIL
- **预期行为**: 三个 smoke 用例均可按顺序运行，且结果无 panic/fatal/Traceback，也不存在重复计数或假通过。
- **实际行为**:
  - 第三个 smoke 用例无法执行，因为 `empty-tcache` 目录缺失。
  - 第二个 smoke 用例 `cpp/20260211-jemalloc-5-3-0-multithread` 虽返回通过，但验证输出里所有 size 类别均约 200%，明显超过预期对象数。
- **复现步骤**:
  1. 执行 `python3 testdata/run_test.py cpp/20260211-jemalloc-5-3-0-multithread`
  2. 查看 `[Per-Size Validation]` 表。
  3. 可见 `Expected=20000, Found≈40000`，`Ratio≈200%`。
  4. 再执行 `python3 -m unittest tests.test_issue_123_jemalloc_regression.TestIssue123JemallocRegression.test_required_jemalloc_testdata_assets_exist` 观察 `empty-tcache` 目录缺失。
- **可能原因**:
  - 多线程统计存在重复计数；或
  - `validate.py` 仅校验“下限 >=99%”，没有校验“上限不能明显超出预期”，导致假通过。
- **建议修复**:
  - 排查多线程 size bucket 聚合逻辑是否把弱引用、别名对象或多个来源重复累计。
  - 收紧多线程 `validate.py` 判定逻辑，增加上限约束，避免 200% 这类结果被判定为 PASS。
  - 补齐 `empty-tcache` 资产后重跑完整 smoke。
