## Summary
Fix ClassifyMemoryPiece weak reference duplicate counting that caused jemalloc 5.1.0/5.2.0/5.2.1 multithread small size buckets (16B-1024B) to show ~200% ratios. Update dev-log to reflect that 4.5.0 multithread 1MB bucket is already fixed to 100%.

## Changes
- `mallocer/pieces.go`: Fix ClassifyMemoryPiece to only create separate (weak) result entries when the weak node has a real C++ type (TypeIdx != 0). When TypeIdx == 0 (only reference chain info, no type inference), register the node in CppPtrMap for reference graph but do not create a duplicate result entry that would be double-counted with InitAllMemoryPieceSize's initial malloc(size) entry.
- `dev-log/2026-03-12-jemalloc-regression-results.md`: Update 4.5.0 multithread 1MB status from "未修复 200%" to "已修复 100%". Update AC status. Add new ClassifyMemoryPiece fix details. Update未完成项 and 未验证项.
