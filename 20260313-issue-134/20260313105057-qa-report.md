## 失败的 Test Case

### TC-005: 不引入双全局状态，并保留 V2 关键能力验证入口
- **结果**: FAIL
- **预期行为**: `go test -race ./common/mpm/...` 或等价命令应真实执行至少一个覆盖 V2 known 位并发路径的测试，从而验证并发安全行为，而不是仅有测试文件存在。
- **实际行为**: `common/mpm/v2/race_test.go` 只有 `testKnownBitmapRaceSmoke`，函数名不是 `Test...`，不会被 Go 测试框架收集执行；因此 race 冒烟命令虽然返回成功，但实际没有跑到任何 V2 测试逻辑。
- **复现步骤**:
  1. 在仓库根目录执行 `go test -race ./common/mpm/...`
  2. 观察输出：`maze/common/mpm/v2` 显示 `[no tests to run]`
  3. 再执行 `go test -race ./common/mpm/v2 -run Test -count=1`
  4. 仍显示 `[no tests to run]`
  5. 打开 `common/mpm/v2/race_test.go`，可见仅定义 `func testKnownBitmapRaceSmoke(t *testing.T)`
- **可能原因**: race 测试文件已创建，但测试函数命名未遵循 Go 测试约定，导致 QA 要求中的 race 验证只在“文件存在性”层面成立，在“实际执行”层面缺失。
- **建议修复**: 将 race 冒烟逻辑暴露为可被 `go test` 识别的 `Test...` 用例，并确保 `go test -race ./common/mpm/...` 输出中不再出现 `maze/common/mpm/v2 [no tests to run]`。
