## Summary

修复了 code review 指出的两个问题：一是 `SetVMType()` / `initVirtualMemory()` 现在会显式校验 VM 绑定结果，避免初始化后请求类型与实际绑定类型静默背离；二是 `copyResultFiles()` 改为流式拷贝，避免复制 `go.cpu` 等较大结果文件时整文件读入内存。

结论：**COMPLETE**

## Changes

- `common/vm.go`: 将 `SetVMType()` 收敛为显式绑定流程，新增目标类型解析与绑定校验；当请求无法生效时直接记录日志并 panic；`initVirtualMemory()` 在 `vm.Init()` 前再次校验当前绑定类型与选型一致。
- `common/vm/facade.go`: 补充静态门面的生命周期注释，明确未初始化 / 已初始化 / 已关闭三个阶段允许的调用边界，降低兼容层误用风险。
- `common/vm/facade_lifecycle_test.go`: 新增初始化后重复选择当前类型返回 true、切换到其他类型返回 false 的测试，覆盖 `SelectImplementation()` 新语义。
- `common/vm_init_integration_test.go`: 恢复旧 VM 类型时显式断言重新绑定成功，避免测试清理阶段吞掉生命周期错误。
- `main.go`: 新增 `copyFile()`，将 `copyResultFiles()` 和日志复制流程改为 `io.Copy` 流式拷贝，保留 `go.cpu` 复制能力并降低内存峰值。
