## Summary
修复了 code review 提出的两个问题：VM facade 热路径不再按值复制整个 implementation 结构体，改为直接读取绑定对象指针；同时修正 vm_v3 集成测试的清理顺序，确保测试结束后真正恢复原有全局状态。

## Changes
- `common/vm/facade.go`: 将 `currentImplementation()` 改为返回绑定实现指针，避免 `GetMemory*` 与 `VMStat` 热路径复制整组函数指针；`dispatchClose()` 改为基于关闭前快照执行关闭，避免清空绑定后丢失 Close 目标。
- `common/vm/init_v3_integration_test.go`: 先执行 `dispatchClose()` 清理测试内状态，再恢复旧 binding / VMType / initialized 标志，消除测试结束后全局状态被错误清空的问题。
