## Summary

修复了代码审查指出的两个阻塞问题：一是收紧 `common/vm` 生命周期 API 语义，明确 `SelectImplementation` 仅允许在初始化前选型，`Init` / `Close` 仅用于初始化与 teardown 阶段；二是撤销 `isPrivateMmap()` 的静默降级，恢复缺失映射时显式失败，并补齐 `vm_v3` 集成测试所需的最小 maps 输入。

## Changes
- `common/vm/facade.go`: 新增初始化状态标记，阻止初始化后的再次选型；把 `Init` / `Close` 改为显式函数并补充单线程生命周期注释。
- `common/vm/facade_lifecycle_test.go`: 新增初始化前后选型约束、初始化标记与关闭后重选的回归测试。
- `common/vm/shared/helpers.go`: 恢复 `isPrivateMmap()` 缺失映射时的 panic 语义，避免将异常输入静默当作非 private mmap。
- `common/vm_init_integration_test.go`: 通过 ELF section 动态构造测试 maps，避免依赖放宽生产语义。
- `dev-log/2026-03-17-vm-facade-no-runtime-switch-plan.md`: 同步记录 API 语义收紧与测试修正方案。
