## 失败的 Test Case

### TC-005: V3 与 V2 对比基准存在且可执行
- **结果**: FAIL
- **预期行为**: `common/vm/bench_compare_test.go` 相关 V3 对比测试全部通过，且 V3 在所有场景优于 V2。
- **实际行为**: compare benchmark 已可运行，但仅并发访问场景优于 V2；随机访问、顺序扫描、4/8 字节读场景均落后。
- **复现步骤**:
  1. 执行 `go test -tags vm_v3 ./common/vm -run TestV3CompareBenchmarksPresent -bench 'BenchmarkCompareV3(GetMemory8|GetMemory4|SequentialScan|RandomAccess|ConcurrentAccess)$' -benchmem -benchtime=100ms`
  2. 对比 `v2-full` 与 `v3` 子 benchmark 输出
- **可能原因**: V3 热路径仍包含 `segmentAt` / `sort.Search` fallback / `GetMemoryRange` 额外分支成本，导致单线程与顺序/随机访问路径未体现预期优化收益。
- **建议修复**: 聚焦地址命中热路径，减少页表 miss 后的二次查找与跨段读路径分配/拷贝开销，并补充能直接断言 V3 胜出的自动化校验。

### TC-008: 性能目标门槛校验
- **结果**: FAIL
- **预期行为**: 随机访问 <4ns、顺序扫描 <3ns、并发访问 <1.5ns。
- **实际行为**: 实测分别为 `78.11ns/op`、`22.45ns/op`、`53.27ns/op`，显著高于目标。
- **复现步骤**:
  1. 执行 `go test -tags vm_v3 ./common/vm/v3 -run '^$' -bench 'Benchmark(RandomAccess|SequentialScan|ConcurrentAccess)$' -benchmem`
  2. 观察 benchmark 输出中的 `ns/op`
- **可能原因**: 当前 V3 实现虽然补齐了结构和测试覆盖，但核心性能优化不足，特别是随机访问与顺序扫描未达到 issue 预期的 30%-50% 提升。
- **建议修复**: 需要以性能剖析为基础重新审视页表命中、缓存策略与批量 API 热路径，而不是仅补齐基准与接口。
