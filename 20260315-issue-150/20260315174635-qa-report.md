## 失败的 Test Case

### TC-006: Python / Node.js 主链路回归未退化
- **结果**: FAIL
- **预期行为**: Python / Node.js 回归集合全部通过，至少应完成代表性与综合回归验证。
- **实际行为**: 综合 Node.js 用例 `testdata/nodejs/20260211-comprehensive` 无现成 tarball；我按 QA 规范自行创建测试数据时，`python3 cmd/maze-gen-coredump.py -o testdata/nodejs/20260211-comprehensive/ "node --expose-gc testdata/nodejs/20260211-comprehensive/test.js"` 在打包阶段失败。
- **复现步骤**:
  1. 运行上述 `maze-gen-coredump.py` 命令。
  2. 等待脚本启动 Node.js 进程并输出 `READY FOR GCORE`。
  3. 观察 `[4/5] Packaging coredump...` 阶段失败。
- **可能原因**: `gcore` 产物 `/tmp/maze-coredump-1898091.1898091` 被 `file` 识别为 `data` 而非可识别 core dump，导致 `maze-tar-coredump.py` 无法解析程序路径；可能与当前环境的 core dump 生成格式或 `maze-tar-coredump.py` 的识别兼容性有关。
- **建议修复**: 排查 `cmd/maze-gen-coredump.py` / `maze-tar-coredump.py` 对当前环境下 gcore 产物的兼容性，确保 QA 可稳定生成 `nodejs/20260211-comprehensive` tarball 后再补跑完整 Node.js 回归。
