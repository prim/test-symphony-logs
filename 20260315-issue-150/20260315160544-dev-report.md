## Summary

本轮为 **PARTIAL** 修复。

已修复 QA 指出的一个关键缺口：在 `common/vm/bench_compare_test.go` 中新增可失败的 V3 vs V2 自动化性能守护测试，不再只有 benchmark 名称存在性检查。这样当 V3 未优于 V2 时，`go test` 会直接失败，能够真实反映 AC9 当前未满足的状态。

同时补跑了 Python / Node.js 主链路回归，确认当前修改未引入这两条链路的功能退化。

## Changes

- `common/vm/bench_compare_test.go`: 新增 `TestV3ComparePerformanceGuard`，覆盖 `GetMemory8`、`GetMemory4`、`SequentialScan`、`RandomAccess`、`ConcurrentAccess` 五个主场景的 V2/V3 性能比较，并在 V3 不优于 V2 时直接失败；补充辅助计时与并发执行函数。
