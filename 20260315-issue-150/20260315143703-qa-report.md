## 失败的 Test Case

### TC-005: V3 与 V2 对比基准存在且可执行
- **结果**: FAIL
- **预期行为**: `common/vm/bench_compare_test.go` 中的 V3 vs V2 对比测试全部通过，且 V3 在所有场景优于 V2。
- **实际行为**: compare benchmark 可运行，但除 `RandomAccess` 外，V3 在 `GetMemory8`、`GetMemory4`、`SequentialScan`、`ConcurrentAccess` 等关键场景均慢于 V2。
- **复现步骤**:
  1. 在仓库根目录执行 `go test -tags vm_v3 ./common/vm -run TestV3CompareBenchmarksPresent -bench 'BenchmarkCompareV3(GetMemory8|GetMemory4|SequentialScan|RandomAccess|ConcurrentAccess)$' -benchmem -benchtime=20ms`
  2. 对比各子 benchmark 的 `v2-full` 与 `v3` 的 `ns/op`
- **可能原因**: 当前 V3 页表/缓存/读取热路径虽已具备框架，但热点路径仍存在额外查找与分支成本，未形成相对 V2 的稳定优势。
- **建议修复**: 继续针对 `lookupSegment`、页表命中路径、顺序扫描路径与并发访问路径做热路径优化，并补充自动断言，避免 compare benchmark 仅“可运行”而不校验“优于 V2”。

### TC-008: 性能目标门槛校验
- **结果**: FAIL
- **预期行为**: 随机访问 `< 4ns`、顺序扫描 `< 3ns`、并发访问 `< 1.5ns`。
- **实际行为**: 随机访问 `21.08 ns/op`，顺序扫描 `18.22 ns/op`，并发访问 `0.7644 ns/op`。
- **复现步骤**:
  1. 在仓库根目录执行 `go test -tags vm_v3 ./common/vm/v3 -run '^$' -bench 'Benchmark(RandomAccess|SequentialScan|ConcurrentAccess)$' -benchmem -benchtime=50ms`
  2. 查看输出中的 `ns/op`
- **可能原因**: 当前 V3 设计已落地，但真实热路径仍未达到设计文档预期；尤其顺序扫描与随机读仍明显慢于目标值。
- **建议修复**: 继续优化地址定位与 segment 命中路径，重新评估页表粒度、L1 cache 命中收益与批量/顺序访问专用路径。
