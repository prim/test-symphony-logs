## 失败的 Test Case

### TC-005: V3 与 V2 对比基准存在且可执行
- **结果**: FAIL
- **预期行为**: V3 vs V2 对比测试通过，且 V3 在所有场景优于 V2。
- **实际行为**: 对比基准虽存在且可执行，但 V3 在 `GetMemory8`、`GetMemory4`、`SequentialScan`、`RandomAccess`、`ConcurrentAccess` 中全部慢于 V2。
- **复现步骤**:
  1. 在仓库根目录执行 `go test -tags vm_v3 ./common/vm -run TestV3CompareBenchmarksPresent -bench 'BenchmarkCompareV3(GetMemory8|GetMemory4|SequentialScan|RandomAccess|ConcurrentAccess)$' -benchmem -benchtime=20ms`
  2. 观察各子基准的 `v2-full` 与 `v3` `ns/op`
- **可能原因**: V3 热路径仍存在额外固定成本；另外对比基准输出中 V3 出现 `B/op` 非零，说明仍可能存在隐藏的运行时开销或测试治具差异。
- **建议修复**: 继续审查 `lookupSegment`、页表命中路径、跨段回退、批量/普通读取共用路径中的额外成本，并核查对比基准环境是否完全对齐 V2 与 V3。

### TC-008: 性能目标门槛校验
- **结果**: FAIL
- **预期行为**: 随机访问 `< 4ns`、顺序扫描 `< 3ns`、并发访问 `< 1.5ns`，且相对 V2 达到目标提升比例。
- **实际行为**: 随机访问 `21.12 ns/op`、顺序扫描 `18.48 ns/op`，远高于阈值；并发访问虽为 `0.7557 ns/op`，但对比基准中 V3 仍慢于 V2。
- **复现步骤**:
  1. 在仓库根目录执行 `go test -tags vm_v3 ./common/vm/v3 -run '^$' -bench 'Benchmark(RandomAccess|SequentialScan|ConcurrentAccess)$' -benchmem -benchtime=50ms`
  2. 查看 `ns/op`
  3. 再执行对比 benchmark 核查相对 V2 表现
- **可能原因**: 当前 V3 实现虽具备页表和 L1 cache，但未将性能优势真正转化为主链路收益；`segmentAt`/`segmentIndexAt` 仍保留二分与跨层逻辑，且缓存命中模式可能不足以覆盖基准访问分布。
- **建议修复**: 重新剖析基准热点，重点验证页表命中是否真正绕开二分路径、L1 cache 命中率是否足够，以及对比基准中 `B/op` 非零来源。
