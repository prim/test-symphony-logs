## Summary

- 修复 `common/vm/v3` 的并发加载热路径，将 segment 数据缓存从 `atomic.Pointer[[]byte]` 调整为单次发布后的直接切片读取，减少稳态读取开销。
- 调整页表命中路径，避免子页命中后的重复 `contains` 校验与结构体拷贝，保持现有功能测试与主链路回归通过。
- 结论仍为 **PARTIAL**：虽然修复了部分热路径与基准夹具问题，但 QA 报告中的性能目标（AC3/AC4/AC5）和 V3 全场景优于 V2（AC9）依旧未满足。

## Changes

- `common/vm/v3/vm.go`: 将 segment 已加载数据改为直接切片缓存，减少稳态读取时的原子指针解引用；保留 CAS 控制的首访加载流程。
- `common/vm/v3/pagetable.go`: 页表查找改为直接使用 entry 指针，子页命中后直接返回 segment，减少热路径分支与复制成本。
- `common/vm/v3/nocopy.go`: 扩大 L1 cache 槽位并按子页粒度计算索引，为后续进一步性能优化保留更细粒度桶位。
