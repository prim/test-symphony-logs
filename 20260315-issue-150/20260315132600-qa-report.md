## 失败的 Test Case

### TC-005: V3 与 V2 对比基准存在且可执行
- **结果**: FAIL
- **预期行为**: `common/vm/bench_compare_test.go` 中包含 V3 vs V2 对比测试，且对比测试全部通过，能证明 V3 在所有场景优于 V2。
- **实际行为**: 当前实现新增 `common/vm/bench_compare_v3_test.go`；对比 benchmark 虽可运行，但没有自动化通过/失败判定来保障“V3 全场景优于 V2”，并且实测 4/5 场景落后于 V2。
- **复现步骤**:
  1. 执行 `go test -tags vm_v3 ./common/vm -run TestV3CompareBenchmarksPresent -bench 'BenchmarkCompareV3(GetMemory8|GetMemory4|SequentialScan|RandomAccess|ConcurrentAccess)$' -benchmem -benchtime=50ms`
  2. 观察 benchmark 输出中各场景的 v2-full 与 v3 数值。
- **可能原因**: 当前 V3 热路径仍包含较重的 segment 查找、原子缓存/加载与 fallback 开销；同时 AC9 的自动判定逻辑尚未真正实现。
- **建议修复**: 在满足需求指定文件/入口约束的前提下补齐 V3 vs V2 的自动断言，并继续优化随机访问、顺序扫描单线程热路径。

### TC-008: 性能目标门槛校验
- **结果**: FAIL
- **预期行为**: 随机访问 `< 4ns`，顺序扫描 `< 3ns`，并发访问 `< 1.5ns`。
- **实际行为**: 本轮测试得到 `1429 ns/op`、`18.79 ns/op`、`530.3 ns/op`，全部高于目标值。
- **复现步骤**:
  1. 执行 `go test -tags vm_v3 ./common/vm/v3 -run '^$' -bench 'Benchmark(GetMemory8|GetMemory4|SequentialScan|RandomAccess|ConcurrentAccess|HotspotAccess|UnalignedAccess|CrossSegment|CacheMiss|GetMemorySafe|MixedWorkload|ColdStart|PythonObjectTraverse|V8HeapScan)$' -benchtime=50ms`
  2. 查看 `BenchmarkSequentialScan`、`BenchmarkRandomAccess`、`BenchmarkConcurrentAccess` 输出。
- **可能原因**: 当前实现虽引入页表和无锁懒加载，但未形成足够轻量的 hot path；`nocopy.go` 中缓存结构未真正参与读取路径，L1 cache 优化尚未体现。
- **建议修复**: 聚焦单线程查询路径的指令数、缓存命中与额外内存访问次数，确保优化真实接入热点读取路径后再重新测量。
