## Summary

PARTIAL：针对 QA 指出的 V3 热路径常数开销问题，新增线性连续 span 快速路径，在连续虚拟地址且底层数据源支持连续读取时，绕过页表查找与逐 segment 懒加载，降低单点读取与顺序扫描开销；同时保留原有页表、L1 cache 与竞态安全逻辑。

## Changes
- `common/vm/v3/vm.go`: 增加 `linearSpan` 连续地址快速路径；在 VM 初始化时识别可线性映射的连续 segment；在 `GetMemoryRange`、`readUint64`、`readUint32` 中优先走直接偏移读取，减少热路径分支与 segment 查找开销。
- `common/vm/v3/test_helpers.go`: 为 benchmark/test 数据源声明是否支持连续 span，确保测试固件可覆盖该优化而延迟加载竞态测试仍保持原语义。
