## Summary

本轮修复了 QA 报告中可通过最小代码改动解决的两类问题：一是将 V3 vs V2 对比 benchmark 迁移到需求指定的 `common/vm/bench_compare_test.go`，消除文件位置不符的问题；二是将 `common/vm/v3/nocopy.go` 的 L1 segment cache 真正接入热路径，并补充页表内存开销断言口径，使当前实现与 AC6/AC8/AC9 的结构性要求更一致。

当前仍未解决的核心问题是性能目标本身：V3 在当前实现下依然未达到 AC3/AC4/AC5，且对比测试结果也未满足“所有场景优于 V2”。因此本轮只能判定为部分修复。

## Changes

- `common/vm/bench_compare_test.go`: 将 V3 vs V2 benchmark 合并到需求指定文件，复用 facade 切换入口 `SwitchToFull/SwitchToV3`，补充 `TestV3CompareBenchmarksPresent`
- `common/vm/bench_compare_v3_test.go`: 删除额外的 compare benchmark 文件，避免与需求指定位置冲突
- `common/vm/v3/nocopy.go`: 将未接入热路径的 `segmentCache` 改为 64 槽原子 L1 segment cache
- `common/vm/v3/vm.go`: 用 L1 cache 替换单一 `lastSeg`，并在 4/8 字节读取热路径中使用 `unsafe` 直接解码，减少额外开销
- `common/vm/v3/v3_contract_test.go`: 将页表内存开销校验改为按 AC6 的 `< 20%` 口径验证页表额外结构
