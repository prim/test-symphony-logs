## 失败的 Test Case

### TC-005: V3 与 V2 对比基准存在且可执行
- **结果**: FAIL
- **预期行为**: `common/vm/bench_compare_test.go` 中的 V3 vs V2 对比测试应能证明 V3 在所有对比场景优于 V2。
- **实际行为**: 当前只有 benchmark 名称存在性检查，没有性能优劣断言；实际跑出的基准结果显示 V3 在所有已测对比场景均慢于 V2。
- **复现步骤**:
  1. 执行 `go test -tags vm_v3 ./common/vm -run TestV3CompareBenchmarksPresent -bench 'BenchmarkCompareV3(GetMemory8|GetMemory4|SequentialScan|RandomAccess|ConcurrentAccess)$' -benchmem -benchtime=20ms`
  2. 对比 `v2-full` 与 `v3` 子 benchmark 的 `ns/op`
  3. 观察到 V3 全部慢于 V2
- **可能原因**: 当前 V3 仍存在显著热路径常数开销；此外，对比测试未落实强约束断言，导致 AC9 只能“能跑”而不能“验收通过”。
- **建议修复**: 继续优化热路径，并补齐可以稳定判定“V3 优于 V2”的比较测试逻辑，而不是仅检查 benchmark 名称存在。

### TC-008: 性能目标门槛校验
- **结果**: FAIL
- **预期行为**: 随机访问 `< 4ns`、顺序扫描 `< 3ns`、并发访问 `< 1.5ns`。
- **实际行为**: 随机访问 `14.89 ns/op`、顺序扫描 `12.79 ns/op`、并发访问 `0.5264 ns/op`。
- **复现步骤**:
  1. 执行 `go test -tags vm_v3 ./common/vm/v3 -run '^$' -bench 'Benchmark(GetMemory8|GetMemory4|SequentialScan|RandomAccess|ConcurrentAccess|HotspotAccess|UnalignedAccess|CrossSegment|CacheMiss|GetMemorySafe|MixedWorkload|ColdStart|PythonObjectTraverse|V8HeapScan)$' -benchmem -benchtime=50ms`
  2. 查看 `BenchmarkSequentialScan` 与 `BenchmarkRandomAccess` 的 `ns/op`
  3. 与 issue #150 阈值比较
- **可能原因**: 新增 `linearSpan` 快速路径后仍无法消除主要常数开销；从对比结果看，V3 热路径仍慢于 V2，说明页表/L1 cache/接口分派的整体成本尚未压低到目标范围。
- **建议修复**: 继续剖析随机访问与顺序扫描热路径，定位比 V2 更慢的具体分支/加载链路，并在对比 benchmark 中验证改动效果。
