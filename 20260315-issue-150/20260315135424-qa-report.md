## 失败的 Test Case

### TC-005: V3 与 V2 对比基准存在且可执行
- **结果**: FAIL
- **预期行为**: `common/vm/bench_compare_test.go` 中 V3 vs V2 对比测试应能证明 V3 在所有场景优于 V2。
- **实际行为**: 实测多个场景 V3 均慢于 V2，且自动化测试未把“全面更优”设为断言。
- **复现步骤**:
  1. 在仓库根目录执行 `go test -tags vm_v3 ./common/vm -run TestV3CompareBenchmarksPresent -bench 'BenchmarkCompareV3(GetMemory8|GetMemory4|SequentialScan|RandomAccess|ConcurrentAccess)$' -benchtime=20ms -benchmem`
  2. 比较各子 benchmark 的 `v2-full` 与 `v3` ns/op。
- **可能原因**: V3 热路径仍存在显著固定成本；同时 AC9 自动化验收并未真正约束性能优于 V2。
- **建议修复**: 补充稳定的对比断言，并继续剖析 V3 读取热路径，确保至少核心场景先稳定快于 V2。

### TC-008: 性能目标门槛校验
- **结果**: FAIL
- **预期行为**: 随机访问 `< 4ns`、顺序扫描 `< 3ns`、并发访问 `< 1.5ns`。
- **实际行为**: 本轮实测分别为 `5572 ns/op`、`19.33 ns/op`、`8133 ns/op`。
- **复现步骤**:
  1. 在仓库根目录执行 `go test -tags vm_v3 ./common/vm/v3 -run '^$' -bench 'Benchmark(RandomAccess|SequentialScan|ConcurrentAccess)$' -benchmem -benchtime=50ms`
  2. 记录输出的 ns/op 指标。
- **可能原因**: 页表命中、segment 查询、lazy load 稳态读取或跨层封装仍存在较大额外成本；当前实现与设计中 L1 cache 优化也未真正接入。
- **建议修复**: 使用 profiling 聚焦 `GetMemory`/`GetMemory4`/`segmentAt` 热路径，并决定接通或移除未生效的 L1 cache 设计层。
