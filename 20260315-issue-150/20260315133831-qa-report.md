## 失败的 Test Case

### TC-005: V3 与 V2 对比基准存在且可执行
- **结果**: FAIL
- **预期行为**: `common/vm/bench_compare_test.go` 中的 V3 vs V2 对比测试应通过，且 V3 在所有场景优于 V2。
- **实际行为**: 对比基准虽然存在并可运行，但 V3 在已执行的 5 个场景中全部慢于 V2。
- **复现步骤**:
  1. 在仓库根目录执行 `go test -tags vm_v3 ./common/vm -run TestV3CompareBenchmarksPresent -bench 'BenchmarkCompareV3(GetMemory8|GetMemory4|SequentialScan|RandomAccess|ConcurrentAccess)$' -benchmem -benchtime=50ms`
  2. 观察 `v2-full` 与 `v3` 子 benchmark 输出
  3. 对比各场景 `ns/op`
- **可能原因**: V3 页表/L1 cache/批量接口虽然已接线，但热路径仍存在额外查找与数据装载开销，未形成对 V2 的性能优势。
- **建议修复**: 继续针对 `segmentAt/lookupSegment/readUint64/readUint32` 热路径做剖析优化，并补齐真正能自动判断“V3 全场景优于 V2”的对比校验，而不是仅验证 benchmark 入口存在。

### TC-008: 性能目标门槛校验
- **结果**: FAIL
- **预期行为**: V3 随机访问 `< 4ns`、顺序扫描 `< 3ns`、并发访问 `< 1.5ns`，并较 V2 达到 issue 指定提升比例。
- **实际行为**: 当前环境下测得 `SequentialScan 18.50 ns/op`、`RandomAccess 10039 ns/op`、`ConcurrentAccess 823.2 ns/op`，均明显偏离目标阈值。
- **复现步骤**:
  1. 在仓库根目录执行 `go test -tags vm_v3 ./common/vm/v3 -run '^$' -bench 'Benchmark(RandomAccess|SequentialScan|ConcurrentAccess)$' -benchmem -benchtime=50ms`
  2. 记录输出的 `ns/op`
  3. 与 issue #150 中 AC3/AC4/AC5 目标对比
- **可能原因**: 当前 benchmark 测试夹具与热路径实现之间存在较大固定成本；尤其 `BenchmarkRandomAccess` / `BenchmarkConcurrentAccess` 结果异常高，说明测试路径仍可能落入慢路径或包含额外内存装载/查表成本。
- **建议修复**: 先对 V3 benchmark 夹具和读取路径做 profiling，定位高开销点，再针对随机访问和顺序扫描路径优化页表命中与 segment cache 使用方式。
