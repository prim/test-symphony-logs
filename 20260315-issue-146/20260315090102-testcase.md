**prim/maze#146**: 删除多余的 --run 选项

## 问题描述

`--run` 选项在当前实现中已经多余，可以删除。

## 当前逻辑

```go
if BuildMode {
    must(doBuild())
    if !RunAfterBuild && !hasAnalysisIntent() {
        os.Exit(0)
    }
}

func hasAnalysisIntent() bool {
    if TarPath != "" || Pid != "" {
        return true
    }
    return IsPostman() || IsPostmanProfile() || IsStandAlone()
}
```

## 为什么多余

1. 实际使用中总是会指定 `--pid` 或 `--tar` 参数
2. 有这些参数时，`hasAnalysisIntent()` 返回 true，即使没有 `--run` 也会继续执行
3. 只有 `./maze --build` 这种单独编译的场景才需要 `--run`，但这种用法很少见

## 建议方案

删除 `--run` 选项，简化逻辑为：

```go
if BuildMode {
    must(doBuild())
    if !hasAnalysisIntent() {
        os.Exit(0)
    }
}
```

- `main.go`: 删除 `--run` flag 定义和 `RunAfterBuild` 变量使用
- `common/var.go`: 删除 `RunAfterBuild` 变量定义
- 文档和测试脚本中的 `--run` 引用需要清理
