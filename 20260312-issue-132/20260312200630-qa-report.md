## 修复策略验证

开发的修复策略是将多条主流程中对"可能无效的内存地址"的非 safe 读取接口（`Pointer()`、`Uint8()`、`Int32()` 等）替换为 safe 版本（`GetMemorySafe()`、`GetMemory4Safe()`、`PointerSafe()` 等），在地址无效时通过返回值 `ok=false` 安全跳过，不再触发 `getStaticMemory()` 中的 `debug.Stack()` 打印。

修复涉及 3 条主要链路：
1. Python 3.11 `_PyInterpreterFrame` 遍历路径 → `python/traverse.go`
2. C++ 指针分类路径 → `cpp/base.go`
3. ptmalloc tcache/fastbin 扫描路径 → `mallocer/ptmalloc/init.go`

附加修复：
4. `python/printer.go` — PyStringCache 并发读写锁
5. `python/classifier_tree.go` — ClassifySizeOfTrees 并发读写锁
6. `main.go` — tar 模式日志回拷

此修复策略合理：未删除错误统计入口、未降低日志级别、未吞掉异常、未跳过分析流程。
