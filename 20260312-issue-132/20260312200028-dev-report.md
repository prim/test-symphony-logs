## Summary

修复代码审查发现的两个 major 问题：`traverseInterpreterFrame()` 和 `traverseFrameObjectPy311()` 中 `_PyInterpreterFrame` 与 `PyCodeObject` 字段读取的 safe/unsafe 混用。将所有相关内存读取统一收敛为 safe 版本，消除了在损坏地址上触发 `getStaticMemory()` goroutine 堆栈日志风暴的风险。同时修复了 `walkInterpreterFrames()` 中 code/co_name 的 unsafe 读取，以及 `copyResultFiles()` 静默忽略 I/O 错误的非阻塞建议。

## Changes

- `python/traverse.go`: 
  - `traverseInterpreterFrame()`: 将 `appendPtr` 从 `Pointer(addr)` 改为 `GetMemorySafe(addr)`；将 `PyCodeObject` 字段读取（`Co_nlocals`、`Co_cellvars`、`Co_freevars`）从直接 `Int()`/`Pointer()` 改为 `GetMemory4Safe()`/`GetMemorySafe()`
  - `traverseFrameObjectPy311()`: 将 `Uint8()`、`Int32()`、`Pointer()` 和 `pyCodeObject().Co_*()` 调用全部改为对应 safe 版本
  - `walkInterpreterFrames()`: 将 `Pointer(curFrame + ifCodeOff)` 和 `pyCodeObject(code).Co_name()` 改为 `GetMemorySafe` 链式读取
- `main.go`:
  - `copyResultFiles()`: 为 `ReadFile`/`WriteFile` 失败路径补充 `Debug.Printf` 日志输出
