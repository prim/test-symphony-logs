# Feature: 将 maze-db 和 prim-gdb 目录迁移至 gdb/ 子目录

## 背景与目标
当前项目根目录下存在两个与 GDB 功能紧密相关的目录：
1. **`maze-db/`**：用于存储 Python ctypes 结构缓存数据（约 636 个文件），是 GDB 交互过程中的数据持久化目录
2. **`prim-gdb/`**：包含完整的 GDB 源码和编译后的便携版 GDB 二进制（`gdb-portable`，约 101MB），是一个 git submodule

为了优化项目根目录结构，将与 GDB 相关的资源统一归类管理，需要将这两个目录迁移到 `gdb/` 目录下：
- `maze-db/` → `gdb/maze-db/`
- `prim-gdb/` → `gdb/prim-gdb/`

## 技术方案

```
Before:
./
├── maze-db/          (ctypes cache files)
├── prim-gdb/         (GDB source & portable binary)
└── gdb/

    └── gdb-server/

After:
./
└── gdb/
    ├── gdb-scripts/
    ├── gdb-server/
    ├── maze-db/      (moved from root)
    └── prim-gdb/     (moved from root)
```
