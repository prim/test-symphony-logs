PARTIAL：本轮修复了 VM V3 的两类代码审查阻塞问题，并消除了 ColdStart 对比基准落后于 V2 的问题。具体包括：

## Self-Review

- 发现：`runtimeState.read` 之前是独立堆对象，冷启动路径多一次对象分配与额外指针跳转。修复：将其内联到 `runtimeState` 中，并继续通过原子指针发布只读快照。
- 发现：direct read 快路径在未对齐地址下仍可能直接 `unsafe` 解引用。修复：增加地址对齐检查，不满足时退回 `binary.LittleEndian`。
- 发现：source 重绑后若 `Segment.data` 尚未失效，`fastBase()` 可能复用旧 generation 数据。修复：优先比对 loader generation，仅当 generation 一致时才直接复用缓存。
