## 失败的 Test Case

### TC-005: V3 与 V2 对比基准存在且可执行
- **结果**: FAIL
- **预期行为**: `common/vm/bench_compare_test.go` 中 V3 vs V2 compare suite 全部通过，且 V3 在所有场景优于 V2。
- **实际行为**: `ColdStart` 场景 V3 为 `549282 ns/op`，远慢于 V2 的 `1151 ns/op`。
- **复现步骤**:
  1. 在项目根目录执行：`GOFLAGS='-tags=vm_v3' go test ./common/vm -run TestV3ComparePerformanceGuard -count=1`
  2. 或执行：`GOFLAGS='-tags=vm_v3' go test ./common/vm/v3 ./common/vm -run '^$' -bench 'Benchmark(SequentialScan|RandomAccess|ConcurrentAccess|CompareV3(GetMemory8|GetMemory4|SequentialScan|RandomAccess|ConcurrentAccess|GetMemoryRange|ColdStart))$' -benchmem -benchtime=40ms -count=1`
  3. 观察 `BenchmarkCompareV3ColdStart` 输出。
- **可能原因**: V3 初始化阶段创建 VM、线性 span / 页表 / segment 状态的成本较高，冷启动路径未达到与 V2 同量级。
- **建议修复**: 优先剖析 V3 `Init()` 与 `newVM()` 冷启动热区，定位 `OpenMmap`、linear span 构建、segment 初始化的主要耗时来源。

### TC-008: 性能目标门槛校验
- **结果**: FAIL
- **预期行为**: 随机访问 < 4ns，顺序扫描 < 3ns，并发访问 < 1.5ns，且 V3 对比 V2 在所有场景更快。
- **实际行为**: 顺序扫描为 `3.683 ns/op`，未达到 `< 3ns`；冷启动对比也未满足 V3 优于 V2。
- **复现步骤**:
  1. 在项目根目录执行：`GOFLAGS='-tags=vm_v3' go test ./common/vm/v3 ./common/vm -run '^$' -bench 'Benchmark(SequentialScan|RandomAccess|ConcurrentAccess|CompareV3(GetMemory8|GetMemory4|SequentialScan|RandomAccess|ConcurrentAccess|GetMemoryRange|ColdStart))$' -benchmem -benchtime=40ms -count=1`
  2. 检查 `BenchmarkSequentialScan` 与 `BenchmarkCompareV3ColdStart` 输出。
- **可能原因**: 顺序扫描路径仍存在额外分支或缓存命中不足；冷启动初始化成本过高。
- **建议修复**: 分别针对顺序扫描热路径和冷启动初始化路径做剖析，确认是否还能减少 address check、segment lookup 或初始化时的整体扫描/映射开销。
