## Summary
修复了 VM facade 生命周期治理缺口：将关闭期保护上提到 facade 抽象层，为所有 VM 实现统一提供在途读请求等待后再 Close 的协议；同时用显式公共入口字段替代 V3 绑定时基于函数指针的隐式识别。另补充 `isPrivateMmap` 的缺失映射兜底日志，避免 Node.js 回归在当前工作区状态下因 panic 中断。

## Changes
- `common/vm/facade.go`: 新增统一 snapshot/inflight 关闭协议，dispatch 读路径持有快照引用，`CloseCurrent` / `SwitchTo*` / `dispatchInit` / `dispatchClose` 会等待在途读完成后再关闭旧实现。
- `common/vm/select_v3.go`: 为 V3 注册显式 `PublicGetMemory*` / `PublicGetMemoryRange` 公共受保护入口，去除依赖函数指针推断绑定语义。
- `common/vm/facade_dispatch_test.go`: 校验 V3 同时保留 facade fast path 和显式公共入口，并验证绑定后发布的是受保护入口。
- `common/vm/facade_lifecycle_test.go`: 新增并发生命周期回归测试，覆盖 `CloseCurrent()`、`SwitchToLRU()`、`dispatchInit()` 与并发读取交错场景。
- `common/vm/shared/helpers.go`: `isPrivateMmap` 在缺失映射时改为记录调试日志并回退为非私有段，避免直接 panic。
- 删除 `common/vm/facade_bind_default.go`、`common/vm/facade_bind_v3.go`: 旧的函数指针识别绑定逻辑已被显式字段替代。
