## Changes

- `common/vm/v3/nocopy.go`: 简化 L1 cache 索引函数，压缩随机访问热路径常数开销。

## Self-Review 发现与修复

- 发现关闭阶段仅依赖 `closing` 标记不足以阻止 late reader 进入，因此改为在 `readers` 上引入关闭哨兵值，确保关闭开始后不再接受新 reader。
- 发现 facade 经过通用 `GetMemory` 导出函数时会重复走 `acquireReadState()`，在 compare guard 的稀疏场景中会放大常数成本，因此为 facade 单独导出直接快路径入口并在 `select_v3.go` 绑定。
- 发现页表查询先查 sub table 再查单 segment 直返会给单 segment bucket 增加额外分支，因此调整为优先命中 `entry.seg`。
