## 失败的 Test Case

### TC-008: 性能目标门槛校验
- **结果**: FAIL
- **预期行为**: V3 顺序扫描延迟应 `< 3ns/op`，同时满足 AC3/AC4/AC5 三项性能门槛。
- **实际行为**: 随机访问和并发访问均达标，但 `BenchmarkSequentialScan-48` 实测 `3.956 ns/op`，高于目标阈值。
- **复现步骤**:
  1. 在仓库根目录执行 `./maze --build`
  2. 执行：
     ```bash
     GOFLAGS='-tags=vm_v3' go test ./common/vm/v3 -run '^$' -bench 'Benchmark(SequentialScan|RandomAccess|ConcurrentAccess)$' -benchmem -benchtime=80ms -count=1
     ```
  3. 观察 `BenchmarkSequentialScan` 的 `ns/op`
- **可能原因**: 顺序扫描热路径虽已增加 iterator 对齐快路径与 runtimeState 内联优化，但当前 `GetMemory` 主路径在本机环境上仍未压缩到 `< 3ns/op`；说明线性 span / 快路径命中后的剩余开销仍偏高。
- **建议修复**: 继续针对顺序读取主路径做微优化，并以当前 QA 环境复测，确保 `BenchmarkSequentialScan` 稳定小于 3ns/op，而非仅在单次开发机结果接近阈值。
