## 失败的 Test Case

### TC-005: V3 与 V2 对比基准存在且可执行
- **结果**: FAIL
- **预期行为**: `common/vm/bench_compare_test.go` 中 V3 vs V2 对比测试通过，且 V3 在所有场景优于 V2。
- **实际行为**: `TestV3ComparePerformanceGuard` 在 `GetMemory8PageTable` 场景失败，V3 中位数明显慢于 V2。
- **复现步骤**:
  1. 在仓库根目录执行 `GOFLAGS='-tags=vm_v3' ./maze --build`
  2. 执行 `go test -tags vm_v3 ./common/vm/...`
  3. 观察 `TestV3ComparePerformanceGuard` 输出
- **可能原因**:
  - 虽然本轮修复了公共入口的 reader 生命周期保护与 `Close()` 幂等问题，但 page table 热路径的读取开销仍高于 V2。
  - 当前 facade/公共入口保护逻辑可能引入额外原子操作或间接访问成本；也可能是 V3 page table 查址路径仍未消除关键热点。
- **建议修复**:
  - 优先剖析 `GetMemory8PageTable` 的热路径，区分 facade 开销、reader 保护开销、page table 查址开销、segment load/fastData 命中开销。
  - 修复后重新执行 `go test -tags vm_v3 ./common/vm/...` 与相关 benchmark 对比。
