## 失败的 Test Case

### TC-005: V3 与 V2 对比基准存在且可执行
- **结果**: FAIL
- **预期行为**: `common/vm/bench_compare_test.go` 中的 V3 vs V2 对比测试应证明 V3 在所有关键场景优于 V2。
- **实际行为**: 仅并发访问场景 V3 更快，其余 4 个主场景均慢于 V2。
- **复现步骤**:
  1. 在仓库根目录执行：
     ```bash
     GOFLAGS='-tags=vm_v3' go test ./common/vm ./common/vm/v3 -run '^$' -bench 'Benchmark(SequentialScan|RandomAccess|ConcurrentAccess|CompareV3(GetMemory8|GetMemory4|SequentialScan|RandomAccess|ConcurrentAccess))$' -benchmem -benchtime=40ms -count=1
     ```
  2. 观察 `common/vm` 包下 `BenchmarkCompareV3*` 输出。
- **可能原因**: V3 虽完成无锁与页表实现，但当前 compare benchmark 结果表明其热路径总体仍未优于 V2；同时测试守护逻辑 `requireV3FasterThanV2` 只限制“不要比 V2 慢超过 1.5 倍”，弱于 AC9 的真实要求。
- **建议修复**:
  - 优化 V3 热路径，重点关注随机访问与顺序访问的常数成本。
  - 将 compare 守护条件恢复为真正校验 “V3 优于 V2”，避免测试通过但 AC 未达成。

### TC-008: 性能目标门槛校验
- **结果**: FAIL
- **预期行为**:
  - 随机访问 < 4ns
  - 顺序扫描 < 3ns
  - 并发访问 < 1.5ns
- **实际行为**:
  - 随机访问 `5.287 ns/op`
  - 顺序扫描 `3.952 ns/op`
  - 并发访问 `0.2717 ns/op`
- **复现步骤**:
  1. 在仓库根目录执行：
     ```bash
     GOFLAGS='-tags=vm_v3' go test ./common/vm/v3 ./common/vm -run '^$' -bench 'Benchmark(SequentialScan|RandomAccess|ConcurrentAccess|CompareV3(GetMemory8|GetMemory4|SequentialScan|RandomAccess|ConcurrentAccess))$' -benchmem -benchtime=40ms -count=1
     ```
  2. 读取 `maze/common/vm/v3` 包的 benchmark 输出。
- **可能原因**: 当前 linear span / page table / batch 设计已经工作，但随机路径与顺序路径仍存在额外分派或查找成本，尚未兑现设计文档声明的 30%-50% 提升。
- **建议修复**:
  - 继续压缩 `GetMemory` / `GetMemory4` 主路径指令数。
  - 针对顺序扫描路径分析 span 命中后的剩余开销。
  - 复核 compare fixture 与真实使用路径的一致性，确认 benchmark 没有偏向 V2。
