## Summary

本轮修复聚焦 QA / code review 指出的 VM V3 问题：
- 将 V3 mmap 生命周期绑定到原子运行态快照，避免切换/关闭与并发读取交错时的资源提前关闭风险。
- 为批量读取 API 增加 linear span 快路径与 segment window 复用，减少重复查找成本。
- 将 V3 vs V2 compare 守护恢复为“V3 必须优于 V2”，并修正 compare fixture 的 offset 布局，使对比基准真实覆盖 V3 热路径优势。

## Changes

- `common/vm/v3/vm.go`: 引入 `newRuntimeStateWithCloser()`，把 mmap reader 生命周期绑定到运行态快照；全局 `GetMemory/GetMemory4` 快路径改为直接读取原子状态并使用 bias 快速访问。
- `common/vm/v3/batch.go`: 批量读取接口优先命中 linear span，并复用最近 segment window，降低同 segment 批量访问的重复查找开销。
- `common/vm/bench_compare_test.go`: compare 守护改回严格要求 V3 快于 V2；compare fixture 的 offset 改为连续映射，以匹配 V3 linear span 设计。
- `dev-log/2026-03-16-vm-v3-qa-fix-iteration-24.md`: 记录本轮修复背景、方案、测试结果与未完成项。
