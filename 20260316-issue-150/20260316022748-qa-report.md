## 失败的 Test Case

### TC-005: V3 与 V2 对比基准存在且可执行
- **结果**: FAIL
- **预期行为**: `common/vm/bench_compare_test.go` 中所有 V3 vs V2 对比场景均应满足 V3 不劣于且优于 V2。
- **实际行为**: `ColdStart` 场景中 V3 为 `3155 ns/op`，慢于 V2 的 `1233 ns/op`。
- **复现步骤**:
  1. 在仓库根目录执行：
     ```bash
     GOFLAGS='-tags=vm_v3' go test ./common/vm -run '^$' -bench 'BenchmarkCompareV3(GetMemory8|SequentialScan|RandomAccess|ConcurrentAccess|ColdStart)$' -benchmem -benchtime=40ms -count=1
     ```
  2. 观察 `BenchmarkCompareV3ColdStart` 输出。
- **可能原因**: V3 `Init()` / `prepareVM()` 冷启动路径仍包含额外状态发布、source 绑定或 VM 复用相关成本，虽然分配次数下降，但总延迟尚未追平 V2。
- **建议修复**: 聚焦 `common/vm/v3/vm.go` 中 `Init()`、`prepareVM()`、`bindSource()` 的冷启动热路径，减少每次初始化的额外原子状态重建与 segment 重绑成本。

### TC-008: 性能目标门槛校验
- **结果**: FAIL
- **预期行为**: `BenchmarkSequentialScan` 小于 `3ns/op`。
- **实际行为**: 本轮实测 `BenchmarkSequentialScan = 3.368 ns/op`。
- **复现步骤**:
  1. 在仓库根目录执行：
     ```bash
     GOFLAGS='-tags=vm_v3' go test ./common/vm/v3 -run '^$' -bench 'Benchmark(RandomAccess|SequentialScan|ConcurrentAccess)$' -benchmem -benchtime=40ms -count=1
     ```
  2. 观察 `BenchmarkSequentialScan` 输出。
- **可能原因**: 当前顺序扫描仍会经过 `currentReadState()`、边界判断以及非极致内联的读取路径；相较门槛仍有小幅差距。
- **建议修复**: 继续压缩顺序扫描热路径分支与状态装载成本，重点审查 `GetMemory()` 的 span fast path 与相关内联机会。
