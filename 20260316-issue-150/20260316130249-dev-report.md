## Summary
本轮修复了 V3 closing 状态下旧 fast path 仍可命中的生命周期问题，并补强了纯 `vm_v3` 构建下公共初始化路径的验证；同时将大 core 场景优先切回 mmap source，避免 cold start 因整堆预读放大固定成本。

## Changes
- `common/vm/v3/vm.go`: 为 `runtimeReadState` 始终绑定 owner，新增 `canUseFastReadState()` 收紧快路径命中条件，修复 closing 状态绕过保护的问题；将 `GetMemoryRange` 的拷贝缓冲改为复用 `shared.GetByteSlice()`；将 eager heap source 阈值从 256MiB 下调到 64MiB，降低冷启动整堆预读开销。
