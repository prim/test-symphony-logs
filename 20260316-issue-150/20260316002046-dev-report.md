## Summary

本轮修复了 code review 中指出的三个核心问题：
- 为 `common/vm/facade.go` 引入基于原子快照与生命周期锁的实现切换，避免运行时切换时函数表半更新与 `Close/Init` 交错。
- 为 `common/vm/v3/vm.go` 收敛全局热状态为只读运行时快照，并为未对齐地址读取增加 `binary.LittleEndian` 慢路径，降低平台相关未定义行为风险。
- 为 `common/vm/v3/batch.go` 增加 segment 窗口复用，避免批量 API 退化为完全逐项重复查找。

## Changes

- `common/vm/facade.go`: 用原子快照派发替代直接重绑全局函数指针，并用互斥锁串行化切换/关闭生命周期。
- `common/vm/v3/vm.go`: 移除 `activeVM/hotVM/hotSeg/hotSpan*` 等裸全局热缓存，改为 `runtimeState` 原子快照；未对齐读取走安全慢路径。
- `common/vm/v3/batch.go`: 增加批量读取窗口缓存，复用当前 segment 已加载数据。
- `common/vm/v3/v3_contract_test.go`: 基准测试安装逻辑改为使用运行时快照。
- `common/vm/bench_compare_test.go`: 性能守护改为中位数 + 容忍区间，并直接调用具体实现，避免 facade 切换噪声污染结果。
