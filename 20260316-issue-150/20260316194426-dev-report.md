## Summary

本轮修复了 code review 提出的 3 个问题：
- 为生产 `mmapReaderAdapter.Slice()` 增加边界保护，异常 offset/length 下安全返回 `nil`，避免 slice 越界 panic。
- 移除基于 `testing.Benchmark` 的脆弱性能守护门槛，改为验证 V2/V3 在页表稀疏固件上的结果一致性，降低 CI 噪声敏感度。
- 将 `GetMemoryRange` 对比 benchmark 改为“拷贝语义一致”的比较版本，避免把 V2 零拷贝切片与 V3 安全拷贝直接对比。

## Changes
- `common/vm.go`: 为 `mmapReaderAdapter.Slice()` 增加 nil、越界、溢出保护。
- `common/vm/bench_compare_test.go`: 删除不稳定的微基准强制优胜断言；新增 range 拷贝包装；将 `BenchmarkCompareV3GetMemoryRange` 重命名为 `BenchmarkCompareV3GetMemoryRangeCopied`；把性能守护测试改为结果一致性守护。
- `common/vm_adapter_test.go`: 新增 `mmapReaderAdapter` 边界行为测试。
