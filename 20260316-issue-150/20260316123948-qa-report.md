## 失败的 Test Case

### TC-005: V3 与 V2 对比基准存在且可执行
- **结果**: FAIL
- **预期行为**: `common/vm/bench_compare_test.go` 中 V3 vs V2 对比测试全部通过，且 V3 在所有场景优于 V2。
- **实际行为**: benchmark 可执行，但 V3 在随机读取、顺序扫描、并发访问三个已测核心场景全部慢于 V2，其中并发场景退化最明显（`43.48 ns/op` vs `2.504 ns/op`）。
- **复现步骤**:
  1. 在项目根目录执行 `go test -tags vm_v3 ./common/vm ./common/vm/v3 -run '^$' -bench 'Benchmark(CompareV3GetMemory8|CompareV3SequentialScan|CompareV3ConcurrentAccess|RandomAccess|SequentialScan|ConcurrentAccess)$' -benchtime=200ms -benchmem -count=1`
  2. 观察 `maze/common/vm` 包输出中的 CompareV3 benchmark 结果。
- **可能原因**: 对比基准调用路径可能仍走 facade/保护层，导致 V3 快路径优势未体现；并发路径可能存在额外状态/同步开销，和 AC 目标不一致。
- **建议修复**: 复核 compare benchmark 的测试方法与 V3 热路径绑定方式，确认对比是否公平且走到预期快路径；重点分析并发读取路径中的额外开销来源。

### TC-008: 性能目标门槛校验
- **结果**: FAIL
- **预期行为**: 随机访问 `< 4ns`，顺序扫描 `< 3ns`，并发访问 `< 1.5ns`，且相对 V2 达到要求提升比例。
- **实际行为**: 本轮 `BenchmarkRandomAccess` 为 `4.671 ns/op`，超过目标上限；顺序扫描与并发访问达标，但整体 AC3/AC9 仍不成立。
- **复现步骤**:
  1. 在项目根目录执行 `go test -tags vm_v3 ./common/vm ./common/vm/v3 -run '^$' -bench 'Benchmark(CompareV3GetMemory8|CompareV3SequentialScan|CompareV3ConcurrentAccess|RandomAccess|SequentialScan|ConcurrentAccess)$' -benchtime=200ms -benchmem -count=1`
  2. 观察 `maze/common/vm/v3` 包中的 `BenchmarkRandomAccess` 输出。
- **可能原因**: 随机访问快路径性能对 CPU 频率/调度敏感；也可能是页表查找、segment cache 或安全包装层仍有常量级开销未优化到目标范围。
- **建议修复**: 结合 `BenchmarkRandomAccessPageTable`、segment cache 命中路径和 compare benchmark 结果继续做 profiling，确认随机访问是否稳定命中预期快路径。
