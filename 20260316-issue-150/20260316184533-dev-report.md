## Summary
修复了 VM facade 在切换与重初始化窗口内会提前把旧实现标记为 closing、导致并发读请求短暂失败的问题；同时补充平滑切换并发契约测试，并修正 V3 性能守卫测试中噪声容忍条件的判定语义。

## Changes
- `common/vm/facade.go`: 调整 `switchImplementation()` 与 `dispatchInit()` 的生命周期顺序，先完成新实现初始化并发布，再排空并关闭旧实现，保证切换窗口内旧实现仍可读。
- `common/vm/facade_lifecycle_test.go`: 新增切换/重初始化窗口持续可读测试，补充“初始化可先开始、关闭必须等待在途读取排空”的并发契约覆盖，并同步修正旧断言。
- `common/vm/bench_compare_test.go`: 修正 `requireV3FasterThanV2()` 条件，使 `tolerance` 与 `fasterRounds` 真正参与噪声容忍判断。
- `dev-log/2026-03-16-vm-v3-qa-fix-iteration-41.md`: 记录本轮修复背景、方案与验证结果。
