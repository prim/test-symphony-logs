## Summary
本轮修复了 V3 关闭阶段资源释放不及时的问题，并补上 shared static memory 读取的边界检查，避免靠近静态段尾部时发生越界切片或 panic。

## Changes
- `common/vm/v3/vm.go`: `Close()` 改为原子摘除旧运行态后立即关闭旧 mmap reader；移除 `Segment.fastBase()` 中无必要的 `bias` 惰性写入，降低并发读路径数据竞争风险。
- `common/vm/shared/helpers.go`: 为 `GetStaticMemory*` / `GetStaticMemory*Safe` 增加地址溢出、虚拟地址范围和底层切片长度检查，保证静态段尾部访问安全失败而非越界。

## Testing
- `./maze --build`
- `GOFLAGS='-tags=vm_v3' go test ./common/vm/v3 ./common/vm ./common/vm/shared`
- `GOFLAGS='-tags=vm_v3' go test -race ./common/vm/v3 -run TestConcurrentLazyLoadRaceFree -count=1`
- `GOFLAGS='-tags=vm_v3' go test ./common/vm/v3 ./common/vm -run '^$' -bench 'Benchmark(SequentialScan|RandomAccess|ConcurrentAccess|CompareV3(GetMemory8|GetMemory4|SequentialScan|RandomAccess|ConcurrentAccess))$' -benchmem -benchtime=40ms -count=1`
- `python3 testdata/run_test.py python/20260129-complex-types-311`
- `python3 testdata/run_test.py nodejs/20260225-maze-vs-heapsnapshot`

结果：构建、单测、race 测试、Python/Node.js 主链路回归均通过；但 V3 绝对性能门槛仍未满足（顺序扫描约 3.374ns/op，随机访问约 4.851ns/op），因此本轮结论为 PARTIAL。
