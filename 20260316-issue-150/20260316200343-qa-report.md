## 失败的 Test Case

### TC-005: V3 与 V2 对比基准存在且可执行
- **结果**: FAIL
- **预期行为**: `common/vm/bench_compare_test.go` 中 V3 vs V2 对比测试全部通过，且 V3 在所有场景优于 V2。
- **实际行为**: `ColdStart` 对比场景中，V3 远慢于 V2；其余场景虽然领先，但不足以满足 AC9 的“全部场景”要求。
- **复现步骤**:
  1. 在仓库根目录执行 `./maze --build`
  2. 执行：
     ```bash
     go test -tags vm_v3 ./common/vm -run '^$' -bench 'BenchmarkCompareV3(GetMemory8|GetMemory4|SequentialScan|SequentialScanPageTable|RandomAccess|RandomAccessPageTable|ConcurrentAccess|ConcurrentAccessPageTable|ColdStart)$' -benchtime=50ms -count=1
     ```
  3. 观察 `BenchmarkCompareV3ColdStart` 的 v2/v3 指标。
- **可能原因**: 从最新 dev report 可见，当前实现为保留跨次初始化模板缓存做了缓存生命周期调整，但 `ColdStart` 仍出现明显回退，说明初始化/首次访问路径仍存在较高额外成本，模板复用尚未将冷启动成本压低到 V2 之下。
- **建议修复**: 重点分析 `v3.Init()/prepareVM()` 与首个 `GetMemory()` 的冷启动路径，确认模板缓存命中率、页表克隆成本、segment source 初始化成本，以及是否存在每轮 benchmark 重复的大对象分配或缓存失效。
