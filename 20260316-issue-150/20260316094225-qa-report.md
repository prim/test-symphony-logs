## 失败的 Test Case

### TC-005: V3 与 V2 对比基准存在且可执行
- **结果**: FAIL
- **预期行为**: `TestV3ComparePerformanceGuard` 通过，V3 在 page-table 稀疏场景下优于 V2。
- **实际行为**: `GetMemory8PageTable` 对比失败，V3 中位数高于允许阈值。
- **复现步骤**:
  1. 在仓库根目录执行：`go test -tags vm_v3 ./common/vm -run 'TestV3ComparePerformanceGuard|TestV3CompareBenchmarksPresent|TestV3SparseFixturesExerciseNonLinearPaths' -count=1`
  2. 观察 `TestV3ComparePerformanceGuard` 失败输出。
- **可能原因**: V3 在稀疏 page-table lookup / `segmentAt` 热路径常数开销仍偏大，compare guard 中的非线性 fixture 放大了该问题。
- **建议修复**: 继续优化 V3 稀疏页表随机访问热路径，重点检查页表命中后的 segment 定位、缓存命中率与额外分发层开销。

### TC-008: 性能目标门槛校验
- **结果**: FAIL
- **预期行为**: `BenchmarkRandomAccess` < `4ns/op`，且满足 issue 中 30%+ 提升目标。
- **实际行为**: `BenchmarkRandomAccess` 实测 `4.055~4.121 ns/op`。
- **复现步骤**:
  1. 在仓库根目录执行：`go test -tags vm_v3 ./common/vm/v3 -run '^$' -bench 'Benchmark(RandomAccess|SequentialScan|ConcurrentAccess)$' -benchmem -count=3`
