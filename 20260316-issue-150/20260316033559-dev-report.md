## Summary
本轮针对 QA/代码审查反馈做了两项修复：一是为 `MemoryIterator` 增加线性 span 直读路径，减少顺序扫描在连续内存场景下的 segment 切换与重复定位成本；二是将 V3 顺序扫描 benchmark 改为走 iterator 路径，确保批量/顺序扫描 API 的性能收益被实际验证。并复测了关键合约测试、选择/对比测试与核心 benchmark。

## Changes
- `common/vm/v3/batch.go`: 为 `MemoryIterator` 增加 linear 模式，在连续线性 span 上直接顺序读取，避免逐 segment 推进。
- `common/vm/v3/v3_contract_test.go`: 将 `BenchmarkSequentialScan` 改为通过 `MemoryIterator` 执行顺序扫描，并补充迭代器基准辅助函数。
- `dev-flow/plan.md`: 更新 AC9 状态为 PASS；AC4 仍为 FAIL，保留未达标状态说明。

## Testing
- `GOFLAGS='-tags=vm_v3' ./maze --build`
- `GOFLAGS='-tags=vm_v3' go test ./common/vm/v3 -run 'TestConcurrentLazyLoadRaceFree|TestBatchReadAPIContract|TestBatchReadHandlesShortResultsBuffer|TestMemoryIteratorTraversesAcrossSegments|TestPageTableLookupAndFallback|TestGetMemoryRangeHandlesUnalignedAndCrossSegment|TestDirectReadHandlesUnalignedAddress|TestBindSourceInvalidatesStaleSegmentData|TestClosePublishesNewStateBeforeClosingOldReader|TestPageTableMemoryOverhead' -count=1`
- `GOFLAGS='-tags=vm_v3' go test ./common/vm -run 'TestV3ComparePerformanceGuard|TestV3CompareBenchmarksPresent|TestVMV3TypeRegisteredWhenBuiltWithTag|TestSwitchToV3BindsImplementation|TestVMV3BuildTagCompilesAndSelectsV3' -count=1`
- `GOFLAGS='-tags=vm_v3' go test ./common/vm/v3 -run '^$' -bench 'Benchmark(SequentialScan|GetMemory8|RandomAccess|ConcurrentAccess)$' -benchmem -benchtime=80ms -count=1`

关键结果：
- `BenchmarkGetMemory8`: `3.834 ns/op`
- `BenchmarkRandomAccess`: `3.824 ns/op`
- `BenchmarkSequentialScan`: `3.514 ns/op`
- `BenchmarkConcurrentAccess`: `0.2301 ns/op`

结论：AC3/AC5 满足，AC4 仍未达到 `< 3ns/op` 的硬门槛，因此本轮为部分修复。
