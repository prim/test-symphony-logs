## Summary
修复了 V3 生命周期与缓存耦合导致的资源释放问题：`Close()` 现在会清理全局模板缓存，避免最近一次 core 的模板与已加载数据在关闭后继续保活；同时补充回归测试覆盖关闭后缓存清理与重建路径。另对 linear span 直接读取快路径改为基于预计算 `bias` 取指，减少地址换算开销，随机访问基准回到 4ns 附近。

## Changes
- `common/vm/v3/vm.go`: 在 `Close()` 中同步清理 `vmCache`，确保运行态关闭后不再持有最近一次 VM 模板缓存。
- `common/vm/v3/batch.go`: linear span 快路径改为直接使用 `bias` 计算目标指针，减少 `base+offset` 热路径运算。
- `common/vm/v3/v3_contract_test.go`: 新增 `TestCloseClearsTemplateCacheAndRebuildsFreshVM`，验证关闭后缓存释放与重新构建行为。
