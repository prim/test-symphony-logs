## Summary

修复了 VM V3 的 facade 绑定策略：编译期注册继续使用 `v3.Facade*` 快速入口以保留 page table 热路径性能，而运行期真正绑定到当前实现时再切换到 `v3.GetMemory*` / `GetMemoryRange` 的公共受保护入口，从而同时满足 code review 要求的关闭期生命周期保护与 QA 要求的 V3 vs V2 性能守护。

结论：**COMPLETE**

## Changes

- `common/vm/select_v3.go`: 将 VMV3 编译注册入口改为 `v3.Facade*`，避免对比基准误走公共 inflight 包装路径。
- `common/vm/facade.go`: `implementationFor()` 统一通过 `wrapImplementationForBinding()` 生成运行期绑定实现。
- `common/vm/facade_bind_default.go`: 为非 `vm_v3` 构建提供默认 no-op 包装。
- `common/vm/facade_bind_v3.go`: 为 `vm_v3` 构建提供专用包装，仅当实现来源于 `v3.Facade*` 时，在运行期绑定阶段替换为 `v3.GetMemory*` / `GetMemoryRange` 公共保护入口。
- `common/vm/facade_dispatch_test.go`: 调整断言，分别验证“编译期保留 facade fast path”与“运行期绑定发布公共保护入口”两个阶段语义。
