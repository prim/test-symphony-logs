## 失败的 Test Case

### TC-005: V3 与 V2 对比基准存在且可执行
- **结果**: FAIL
- **预期行为**: `common/vm/bench_compare_test.go` 中的 V3 vs V2 对比测试全部通过，且 V3 在所有场景优于 V2。
- **实际行为**: `TestV3ComparePerformanceGuard` 在 `GetMemory8PageTable` 场景失败，V3 中位延迟高于 V2。
- **复现步骤**:
  1. 在仓库根目录执行 `./maze --build`
  2. 执行 `go test -tags vm_v3 ./common/vm -run 'TestV3CompareBenchmarksPresent|TestV3SparseFixturesExerciseNonLinearPaths|TestV3ComparePerformanceGuard' -count=1`
  3. 观察 `TestV3ComparePerformanceGuard` 输出
- **可能原因**: 当前 V3 页表/查址热路径仍存在额外开销，尤其在 `GetMemory8PageTable` 场景下，新的子表实现或读路径封装未带来预期收益。
- **建议修复**: 重点剖析 `common/vm/v3` 中 `segmentAt` / 页表查址 / 公共读入口的指令路径与分支成本，针对 `GetMemory8PageTable` 做更细粒度 profile，对比 V2 热路径的额外原子、边界判断和 fallback 成本。
