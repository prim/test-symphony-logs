## 失败的 Test Case

### TC-005: V3 与 V2 对比基准存在且可执行
- **结果**: FAIL
- **预期行为**: `TestV3ComparePerformanceGuard` 通过，且 V3 在对比场景中稳定优于 V2。
- **实际行为**: `GetMemory8PageTable` 场景稳定失败，7/7 轮中 V3 都慢于 V2。
- **复现步骤**:
  1. 在仓库根目录执行 `./maze --build`
  2. 执行 `go test -tags vm_v3 ./common/vm/...`
  3. 观察 `TestV3ComparePerformanceGuard` 的失败输出
- **可能原因**: 从测试现象看，页表查址主路径或公共读取保护路径仍引入了额外开销；本轮修复引入的生命周期保护虽解决了安全问题，但尚未把 page table 主路径优化回到优于 V2 的水平。
- **建议修复**: 重点分析 `common/vm/v3` 在 page table 命中路径上的额外判断、状态获取与缓存命中代价，并结合 `bench_compare_test.go` 的 sparse fixture 场景复测优化效果。

### TC-008: 性能目标门槛校验
- **结果**: FAIL
- **预期行为**: AC3/AC4/AC5 相关 benchmark 在当前代码与测试环境下可提供稳定达标证据。
- **实际行为**: 关键守护测试已先行失败，无法确认所有性能目标成立。
- **复现步骤**:
  1. 执行 `go test -tags vm_v3 ./common/vm/...`

- **建议修复**: 先修复 AC9 对比守护失败，再重新执行完整 benchmark 套件核对 AC3/AC4/AC5 门槛。
