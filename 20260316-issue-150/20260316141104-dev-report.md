## Summary
修复了最新代码审查指出的两个性能设计问题：批量读取 API 在已排序输入下现在直接走窗口化批处理路径；大 core 初始化不再进入会让公共读入口退化为每次原子引用计数的 mmap reader 路径，从而保留 V3 公共 fast path。

## Changes
- `common/vm/v3/batch.go`: 将已排序输入与未排序输入统一接入批量窗口读取逻辑，仅对未排序输入执行排序，避免已排序扫描场景退化为逐项标量读取。
- `common/vm/v3/vm.go`: 大于 `eagerHeapSourceLimit` 的 core 初始化时改为使用 `fileSource`，避免绑定 `closer` 后禁用公共 fast path。
- `common/vm/v3/v3_contract_test.go`: 增加已排序/未排序 batch workspace 行为测试、大 core 初始化 fast path 守护测试，以及已排序 batch benchmark 入口。
