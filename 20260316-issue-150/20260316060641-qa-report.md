### TC-008: 性能目标门槛校验
- **结果**: FAIL
- **预期行为**: V3 顺序扫描延迟应 `< 3ns/op`，同时维持随机访问 `< 4ns/op`、并发访问 `< 1.5ns/op`。

- **复现步骤**:
  1. 在仓库根目录执行 `./maze --build`
  2. 执行以下命令：
     ```bash
     go test -tags vm_v3 ./common/vm/v3 -run '^$' -bench 'Benchmark(GetMemory8|GetMemory4|SequentialScan|SequentialScanPageTable|RandomAccess|RandomAccessPageTable|ConcurrentAccess|ConcurrentAccessPageTable|HotspotAccess|UnalignedAccess|CrossSegment|CacheMiss|GetMemorySafe|MixedWorkload|ColdStart|PythonObjectTraverse|V8HeapScan)$' -benchtime=80ms -count=1
     ```
  3. 观察 `BenchmarkSequentialScan` 与 `BenchmarkSequentialScanPageTable` 的 ns/op 输出
- **可能原因**: 顺序扫描热路径虽已具备 iterator 与 compare guard，但绝对性能仍受当前 iterator/查址实现开销影响；本轮修复更偏向非线性 fixture 与并发稳定性，未完全消除顺序扫描路径的剩余常数成本。
- **建议修复**: 继续针对顺序扫描热路径做专项优化，并补充更直接的 AC4 守护测试，避免仅依赖 compare guard 而放过绝对阈值回退。
