## Notes for Developer

1. 本 issue 更适合交付一篇结构化分析文档，而不是新增测试文件。
2. 建议文档按“定义 → 核心函数 → 聚合流程 → 上层调用模块 → 特殊分支/并发行为”组织。
3. 建议在文档中列出关键入口文件，至少包括：
   - `common/result_items.go`
   - `python/maze.go`
   - `txos/merge.go`
   - `txos` 目录下若干 `AddToResultItems` 调用点
4. 若要补充“所有上层使用”，建议按“直接调用 / 回调透传 / 局部 resultItems 聚合后 Merge”三类整理，避免简单罗列大量文件。
