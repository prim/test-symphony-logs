## Verification Task

Verify the implementation for:

**prim/maze#147**: VM 门面化重构：拆分为 facade + 版本实现

## 背景

参考 `common/mpm` 和 `common/refs` 的门面化拆分方式，将 VM (Virtual Memory) 重构为稳定门面 + 版本实现结构。

当前 `common/vm.go`、`common/vm_lru.go`、`common/vm_full.go` 承载了两种虚拟内存访问实现：

1. **LRU 模式**: 按需分页缓存，适合大 corefile（>20GB）
2. **FULL 模式**: 一次性加载或 mmap，适合中小 corefile

## 当前问题

- `common/vm.go` 职责过重：既定义接口又承载 ELF 解析、共享库加载等公共逻辑
- 运行时切换依赖 `SetVMType()` 函数，调用方需要关心 VM 类型
- 函数变量全局绑定（`GetMemory = LRU_GetMemory`），不够清晰
- 缺少编译期选择能力，不利于 benchmark 对比和灰度切换
- `common` 包暴露过多 VM 细节

## 目标

1. 建立 `common/vm` 稳定门面，业务层不再直接依赖具体实现
2. 将 LRU 实现沉到 `common/vm/v1` (或 `lru`)
3. 将 FULL 实现沉到 `common/vm/v2` (或 `full`)
4. 保留 `common/vm.go` 作为兼容转发层
5. 支持编译期 build tags 选择实现，同时保留运行时切换能力
