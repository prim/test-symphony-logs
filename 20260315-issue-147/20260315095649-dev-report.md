## Summary

本轮针对最新 code review 提出的两个 major 资源泄漏问题进行了修复：为 VM 门面补充统一的 `Close` 生命周期入口，在运行时切换或重复初始化前释放旧实现资源；同时分别修复了 LRU 实现的 shard 文件句柄泄漏与 FULL 实现的 mmap 泄漏，并补充了重复 `Init()` / 切换场景的回归测试。

结论：**PARTIAL**。本轮已修复最新 QA / code review 指出的问题并完成验证，但原始计划中的 AC4 全量回归与 AC7 性能验证仍未完成。

## Changes

- `common/vm/facade.go`: 为门面实现注册表增加 `Close` 生命周期，切换实现前先关闭当前实现，并暴露 `CloseCurrent()`。
- `common/vm/select_default.go`: 注册 FULL 实现时补充 `Close` 绑定。
- `common/vm/select_dual.go`: 双实现模式下为 `v1` / `v2` 都补充 `Close` 绑定。
- `common/vm/select_full.go`: 注册 FULL 实现时补充 `Close` 绑定。
- `common/vm/select_lru.go`: 注册 LRU 实现时补充 `Close` 绑定。
- `common/vm/shared/helpers.go`: 扩展 `MmapReader` 接口，支持统一关闭 mmap 资源。
- `common/vm/v1/vm.go`: `Init()` 前先执行 `Close()`；新增 shard 文件、缓存状态清理逻辑，避免重复初始化与切换时泄漏 fd。
- `common/vm/v2/vm.go`: `Init()` 前先执行 `Close()`；新增 mmap 关闭逻辑，避免重复初始化与切换时泄漏映射。
- `common/vm.go`: 为兼容层 mmap 适配器补充 `Close()` 转发。
- `common/vm/facade_lifecycle_test.go`: 新增门面切换生命周期测试，验证切换前会调用旧实现 `Close`。
- `common/vm/implementation_lifecycle_test.go`: 新增 v1/v2 重复初始化资源释放回归测试。
