## Self-Review 发现与修复
- 发现 `S.Tcmalloc.Code` 以 `0` 同时表示“旧版本编码”与“未解析”，原始实现用 `S.Tcmalloc.Code == 0` 作为 fallback 条件会覆盖掉已成功识别出的旧版本值。
- 修复方式: 在 `collectRuntimeMetadata()` 中引入局部 `codeResolved` 标志，仅当 size class 签名未解析出结果时才按版本号做 fallback。

## 注意事项
- `parseRevisionFromBinary()` 目前依赖 `strings` 输出中的 revision 模式匹配，适合作为缺失写入路径的保守兜底；若后续出现新格式二进制，可能需要补充更精确的版本提取规则。
- tcmalloc 的 `class_to_size_` 读取依赖 GDB 表达式与 `mallocer/tcmalloc/init.go` 的既有约定，首元素仍需维持 `1` 以兼容当前 size map 初始化逻辑。
