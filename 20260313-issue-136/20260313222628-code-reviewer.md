## 背景
当前 Go 版 prepare 在迁移过程中同时维护了两套结果承载结构：

- 运行期工作上下文：`prepare.Context`
- 下游分析统一读取的全局结果：`common.S`、`common.Cpp`

其中一部分字段在语义、类型和用途上看起来完全等价，只是通过 `prepare/fill.go` 做了一次简单赋值搬运，导致：

- 同一份结果存在双份定义，维护成本高
- `FillCommonVars()` 成为隐式同步点，容易漏改或出现先后顺序问题
- `map` / `slice` 赋值多为浅拷贝，边界不清晰
- `Context` 混合了承载运行态和结果态两类职责，不利于后续继续清理 prepare 架构

目标是把这部分完全等价的结果字段收敛为单一真实来源：**只保留在 `common.S` / `common.Cpp` 中**，不再在 `prepare.Context` 中保留镜像字段，也不再依赖同步搬运。
