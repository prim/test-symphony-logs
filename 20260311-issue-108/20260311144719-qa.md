
# 注意事项

1. **不要删除子模块的提交历史**: `prim-gdb/.git` 实际上是指向 `.git/modules/prim-gdb/` 的链接，移动时要确保这个链接关系正确。

2. **Submodule sync 顺序**: 必须先更新 `.gitmodules`，然后执行 `git submodule sync`，最后提交。

3. **内部配置位置**: Git 在 `.git/modules/` 下维护子模块的完整仓库，路径结构需要与 `.gitmodules` 中的 `path` 对应。

4. **团队协作**: 此修改会影响所有团队成员，迁移完成后需要通知团队执行:
   ```bash

   git submodule sync
   ```
