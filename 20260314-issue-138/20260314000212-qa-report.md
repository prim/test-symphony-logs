### TC-EXT-001: trace replay 查询性能回归检查
- **结果**: FAIL
- **预期行为**: 根据计划文档中的验收建议，组合 workload / trace replay 应继续优于当前基线，至少不应在核心 trace replay query 上落后于 `V1`。

- **复现步骤**:
  1. 在项目根目录执行：
     ```bash
     go test ./common -run ^$ -bench 'BenchmarkMPMSizeAndKnowHit|BenchmarkMPMSizeAndIsKnowMiss|BenchmarkMPMWorkloadMixedRounds|BenchmarkMPMTraceReplayQuery' -benchmem -count=1
     ```
  2. 对比输出中的 `BenchmarkMPMTraceReplayQuery/V1` 与 `BenchmarkMPMTraceReplayQuery/V2`
  3. 可见 `V2` 用时高于 `V1`
- **可能原因**: 当前冻结页范围直达索引与页内小数组线扫对 dense / mixed query-heavy 场景有效，但 trace replay 的访问分布更离散，仍存在页查找与页内定位常数开销未被充分压低。
- **建议修复**: 继续针对 trace replay 真实访问分布做剖析，重点检查页级索引命中模式、range 线性扫描成本，以及页内查找在稀疏/离散地址上的常数开销。
