## 详细结果

### TC-001: facade 删除运行时 implementation 状态
- **结果**: PASS
- **实际行为**: `common/vm/facade.go` 不再声明 `implementationBinding`、`boundImplementation`、`currentBinding()`、`currentImplementation()`、`initialized` 等 facade 运行时实现绑定状态。
- **关键指标**: `go test ./qa/issue162 -run TestFacadeRemovesRuntimeImplementationState` 被 `go test ./qa/issue162 -run Test` 覆盖并通过。
- **备注**: 相关历史兼容状态仅保留在 `_test.go` 测试适配文件中，未进入正式构建路径。

### TC-002: GetMemory 热路径去除 runtime dispatch
- **结果**: PASS
- **实际行为**: `GetMemory*`/`GetMemoryRange`/`VMStat` 在 facade 中直接调用 `implGetMemory*`/`implVMStat`，未再经过 `currentImplementation()` 或 `dispatchGetMemory*()`。
- **关键指标**: `common/vm/facade.go` 中存在 `return implGetMemory(address)` 等直接转发形式；`go test ./qa/issue162 -run TestGetMemoryFacadeStopsRuntimeDispatch` 被总测试覆盖通过。
- **备注**: 满足 AC2、AC4 的结构性要求。

### TC-003: compat 层停止运行时选择并明确重新编译切换
- **结果**: PASS
- **实际行为**: `common/vm.go` 中已删除对 `vm.HasImplementation`、`vm.SelectImplementation`、`vm.CurrentVMType()` 的 compat 运行时选择依赖；`SetVMType()` 仅接受当前编译产物类型或 `VMNone`，否则 panic 并提示需重新编译。
- **关键指标**: `go test ./qa/issue162 -run TestCompatLayerStopsRuntimeSelectionAndDocumentsRebuildOnlySwitch` 被总测试覆盖通过。
- **备注**: 文案与行为均符合“只能通过重新编译切换 VM”的要求。

### TC-004: vm_dual 被删除或降级为非常规模式
- **结果**: PASS
- **实际行为**: `common/vm/select_dual.go` 仍存在，但已明确标注为 `debug / 非常规构建模式`，且不再注册双实现、不再作为常规运行时双实现路径。
- **关键指标**: 未发现 `registerImplementation(` 或旧式 `bindV2()` 常规绑定路径；`go test ./qa/issue162 -run TestVMDualIsRemovedOrExplicitlyDebugOnly` 被总测试覆盖通过。
- **备注**: 满足 AC3。

### TC-005: 各 build tag 选择文件只保留单实现直绑
- **结果**: PASS
- **实际行为**: `select_default.go`、`select_lru.go`、`select_full.go`、`select_v3.go` 均以 `selectedVMType` / `selectedVersion` + `implGetMemory*` 方式进行单实现编译期绑定，不再使用 `registerImplementation()`、`bindV*()`、`currentVersion =` 形式组织运行时实现。
- **关键指标**: `go test ./qa/issue162 -run TestCompileTimeSelectionFilesDirectlyBindSingleImplementation` 被总测试覆盖通过；补充执行 `go test -tags vm_v3 ./common/vm/... ./common` 通过。
- **备注**: 满足“每个编译产物只包含一个 VM 实现”的目标。
