# 拆分 python/classifier.go 的方案设计

**日期**: 2026-03-11

为了提高 `python/classifier.go` (目前近 1750 行) 的可维护性和代码清晰度，我们计划将其按照**职责和模块**拆分为多个更小的源文件。由于这些文件都隶属于 `package python`，内部首字母小写或大写的类型和函数均可互相访问，因此拆分**不需要修改外部包的调用逻辑**。

## 现状分析

`python/classifier.go` 内部目前混合了三类主要逻辑：

1.  **分类数据结构与接口实现**：
    *   各种特定类型的签名特征 `Classify*Type` 结构体，如 `ClassifyArrayType`, `ClassifyMapType`, `ClassifySimpleMapType`, `ClassifyNeteaseType`, `ClassifyMb2ObjectType` 等。
    *   它们实现了排序和特征比较方法：`Len()`, `Less()`, `Swap()`, `T()`, `Reset()`。
2.  **引用树与内存大小分析结构**：
    *   `ClassifyNodeSizeOfTree` 结构体及其实例方法 (`AddNode`, `ToDot`, `ToString`)。
    *   以及对外暴露的核心统计函数 `SizeofTree(pyo uint64, saveFile bool) (string, []byte)`。
3.  **核心调度与具体对象的分类实现逻辑**：
    *   包含了主入口 `classifyPythonObject(...)`。
    *   包含了各个类型特定的分类函数，如 `classifyFrameObject`, `classifyCodeObject`, `classifyDictObject_`, `classifyListObject` 等。这部分占据了整个文件绝大部分的代码行数。

## 推荐的拆分方案（方案二：拆分为 4 个文件）

我们建议将 `python/classifier.go` 拆分为以下 4 个职责单一的文件：

### 1. `python/classifier.go`（主入口与调度）

**职责**：作为对外和对内调度的核心。
**包含内容**：
*   全部的包级别全局变量和常量 (`ClassifyFunctionMap`, `Mutex*`, `Next*Type`，资源池等)。
*   包初始化函数 `initPyClassify()`。
*   主调度入口函数 `classifyPythonObject(...)` 及非常基础的辅助方法（如 `dec(merge)` 等）。

### 2. `python/classifier_types.go`（类型特征提取结构）

**职责**：纯粹存放对象特征计算使用的数据结构与方法。
**包含内容**：
*   `ClassifyLambdaType` 及其方法。
*   `ClassifyArrayType` 及其方法。
*   `ClassifySimpleMapType` 及其方法。
*   `ClassifyMapType` 及其方法。
*   `ClassifyNeteaseType` 及其方法。
*   `ClassifyMb2ObjectType` 及其方法。
*   内部相关的 `TypeMapping` 结构体。

### 3. `python/classifier_tree.go`（内存大小树分析）

**职责**：纯粹处理对象的内存大小遍历与 DOT 格式导出。
**包含内容**：
*   `ClassifyNodeSizeOfTree` 定义。
*   关联的方法如 `AddNode`, `ToDot`, `ToString`。
*   核心函数 `SizeofTree(...)`。

### 4. `python/classifier_objects.go`（对象类型实现集合）

**职责**：最庞大的部分，处理各类 Python 内部对象具体的解析和遍历。
**包含内容**：
*   `classifyFrameObject`, `classifyMethodObject`, `classifyFunctionObject`, `classifyCodeObject` 等。
*   `classifyNoneObject`, `classifyBoolObject`, `classifyIntObject`, `classifyStringObject` 等基础类型。
*   `classifyDictObject_`, `classifyDictObject_3`, `classifyListObject`, `classifySetObjectHelper` 等容器类型。
*   其他的跳过策略函数，如 `classifySkip0`, `classifySkip1`, `IncludePyoRefcntOne`。

## 实施步骤指导

执行拆分时建议采用以下步骤以防遗漏：

1.  创建并移动 `classifier_types.go` 和 `classifier_tree.go`，由于它们依赖和被依赖最少，搬移后不影响主流程。
2.  将所有的 `classify*Object` 剥离到 `classifier_objects.go` 中。
3.  在 `classifier.go` 保留其余的调度代码并修复所需的 import（如 `fmt`, `sync` 等按需引入）。
4.  运行 `go fmt python/*.go` 以标准化格式。
5.  使用 `./maze --build` 进行整体编译，确保没有因私有变量造成的语法或包错误。
6.  执行一次包含完整堆扫描分析的 Python 进程用例（如 `testdata/python/20260129-complex-types-311`），确保运行时 `classifier` 行为正常。
