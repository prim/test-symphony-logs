## Task

Implement the following GitHub Issue:

**prim/maze#156**: Refactor refs API: 删除 Python 专用通道并统一到普通 refs API

## 背景

当前 `refs` 已完成 facade + versioned packages 重构，但仍保留一套 Python 专用 refs API：

- `PythonGetReferrersFast`
- `PythonAddReverseIndex`
- `PythonResetRefs`
- `PythonForEachReverseIndex`
- `PythonReverseIndexStats`

默认绑定也仍是混合模式：

- 普通 refs → `v1`
- Python refs → `v0`

即运行时显示为 `refs=v1+py-v0`。

这会导致：

1. facade 同时维护两组语义重叠的 API
2. Python 上层仍然感知 refs 的内部存储细节
3. Python 自己决定是否遍历 reverse index 并导出到 LevelDB
4. `v0` / `v1` 的职责边界不够清晰
5. 版本切换仍要额外考虑 Python 特殊通道

## 目标

将 Python / Node.js / txos / common 等模块统一到普通 refs API：

- 删除 Python 专用 refs API
- 保留全局版本切换能力：整个 refs 系统选择 `v0` 或 `v1`
- 不再保留 `v1+py-v0` 这种混合绑定
- 不再由 Python 上层决定 refs 是否导出到 LevelDB
- 持久化职责由 refs 后端自行决定

## 方案

统一后的公共 refs API 建议只保留：

```go
AddReverseIndex(from uint64, l []uint64)
AddIndexes(from uint64, l []uint64)
AddIndex(from uint64, l []uint64)

GetReferrers(addr uint64) []uint64
GetReferents(addr uint64) []uint64

Init()
Save()
Check()
```

建议删除：

```go
PythonGetReferrersFast
PythonAddReverseIndex
PythonResetRefs
PythonForEachReverseIndex
PythonReverseIndexStats
ForEachReverseIndex
ReverseIndexStats
ResetRefs
```

关键判断：

- `v0` = 纯内存 refs 后端，不负责持久化
- `v1` = RocksDB refs 后端，持久化和查询由后端负责
- Python 现在那套 `GenerateLevelDBFile()` / reverse-index 导出逻辑属于历史补丁，应该下沉/移除

## 迁移步骤

1. Python 改为只调用普通 `GetReferrers` / `AddReverseIndex`
2. 删除 `PythonResetRefs()`，统一依赖 `Init()` 初始化空状态
3. 删除 facade / compat 层中的 Python 专用 refs 槽位和转发逻辑
4. 删除默认 `v1+py-v0` 绑定，改为单一全局版本
5. 删除 `python/traverse.go` 中基于 reverse index 的 LevelDB 导出逻辑
6. 验证 `refs=v0` / `refs=v1` 下 Python 分析结果都正确

## 兼容性注意

静态排查仓库内代码后，未发现明确读取 Python reverse refs 那批 `maze.db.*` 裸地址 key/value 的仓库内消费者；但正式删除前仍建议确认是否存在 repo 外部脚本依赖。
