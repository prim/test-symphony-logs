## Summary
Implemented the gdbcommand migration by moving the GDB extension module into `gdb-scripts/`, updating the GDB entry script imports, keeping `.maze.py` able to resolve `vtable_get_type_index`, and removing the obsolete root-level release copy step.

## Changes
- `.gdbcommand.py`: added `import gdbcommand` and `from gdbcommand import *` so migrated commands and helpers are loaded from `gdb-scripts/` and exposed in the GDB entry context.
- `.maze.py`: added lazy `get_gdbcommand_module()` resolution and switched the vtable type lookup call to use it.
- `gdb-scripts/gdbcommand.py`: moved the original module here and added fallback definitions for `gdb.TYPE_CODE_*` constants so the lightweight stub-based migration test can import it successfully.
- `maze`: removed the obsolete `cp gdbcommand.py release` release step so packaging only copies GDB scripts from `gdb-scripts/`.
- `dev-log/2026-03-10-move-gdbcommand-to-gdb-scripts.md`: marked completed tasks and acceptance criteria progress.
