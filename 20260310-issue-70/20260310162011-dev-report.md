## Self-Review 发现与修复
- 发现 `.maze.py` 原先依赖 `eval(ps[-3]), eval(ps[-1])` 解析 `pstack` 输出，在 GDB 返回 traceback/错误文本时会崩溃；已改为正则匹配合法 `pthread ... stack begin ... end ...` 行，并跳过异常输出。
- 发现部分 libc 符号表中不存在 `main_arena`/`mp_`，原逻辑会触发 fatal assert；已改为记录 warning 并降级处理，避免中断整个分析流程。
- 发现 `maze-result.json` 在无结果项时输出 `"items": null`，与验证脚本预期的可迭代列表不兼容；已改为稳定输出空数组 `[]`。
