## 详细分析

### GDB Server 进程生命周期问题

`prepare/gdb_server.go` 中 GDB 启动命令：

```go
cmd := exec.Command(
    gdbBin,
    "-q",
    "-n",
    "-x", gdbCommandScript,       // .gdbcommand.py
    "-x", serverScript,            // gdb_socket_server.py
    "-ex", "start_gdb_server 9999",
)
```

GDB 在执行完 `start_gdb_server 9999` 命令后，由于没有后续命令需要执行，且没有交互式输入，GDB 进入"命令执行完毕"状态并退出。虽然 `gdb_socket_server.py` 中的 `start_gdb_server` 启动了 socket 监听线程，但 GDB 主线程退出后，整个进程（包括 daemon 线程）都终止了。

**QA 手动验证过程**：

1. 创建临时目录，创建 gdb 和 .gdbcommand.py 的 symlinks
2. 执行与 `gdb_server.go` 相同的 GDB 命令
3. GDB 日志显示 "GDB Socket Server started on 127.0.0.1:9998"
4. 但 `ps -p $GDB_PID` 显示进程已退出
5. `ss -tlnp | grep 9998` 确认端口未开放

这证实了 GDB 进程在输出启动消息后立即退出的结论。
