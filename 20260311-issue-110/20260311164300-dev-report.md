## 架构变更

### 变更前
```
Python maze 脚本 → .maze.py → maze-py/*.py → .storage.json/.cpp.json → Go .maze
```

### 变更后
```
maze (shell) → .maze (Go binary)
  ├── prepare.Run()  (原 .maze.py 逻辑)
  │    ├── 直接填充 common.S / common.Cpp
  │    └── GDB Socket Client ↔ GDB Server (Python, 保留)
  └── maze1()  (原有分析引擎)
```

## 注意事项

1. **GDB Scripts 保留**: `gdb-scripts/*.py` 和 `gdb/gdb-server/gdb_socket_server.py` 必须继续保留，它们在 GDB 内部运行，使用 GDB Python API

2. **Type 兼容性**: `prepare/fill.go` 中的数据填充使用了与 `common` 包兼容的类型定义，确保数据正确传递

3. **向后兼容**: `common.InitCommon()` 仍然调用 `initConfig()`，但该函数现在仅处理环境变量和基本初始化，不再读取 JSON 文件

4. **Shell 脚本限制**: 新的 `maze` shell 脚本简化了原有 Python 脚本的复杂逻辑（如 --update, --release, --loop 等），这些功能可以通过直接调用 Go 二进制或使用其他工具实现
