## 失败的 Test Case

### TC-007: Python 主流程回归
- **结果**: FAIL
- **预期行为**: `python3 testdata/run_test.py python/20260129-complex-types-311` 成功完成 `--tar` 分析并通过 `validate.py`。
- **实际行为**: `prepare.Run()` 完成后进入 `maze1()`，在 `maze/mallocer/ptmalloc.initPtmallloc()` 读取 `S.Sizeof["struct _heap_info"]` 时触发 panic，测试进程退出码为 2。
- **复现步骤**:
  1. 在仓库根目录执行 `./maze --build`
  2. 执行 `python3 testdata/run_test.py python/20260129-complex-types-311`
  3. 观察终端输出中的 panic 栈
- **可能原因**: `prepare.Run()` 虽然已经完成 GDB 连接与部分符号准备，但未能为 ptmalloc walker 提供 `struct _heap_info` 所需的 `sizeof` 元数据，导致旧的 ptmalloc 初始化逻辑无法继续。
- **建议修复**: developer 需要补齐 prepare 阶段对 ptmalloc 所需 ctypes / sizeof / struct 信息的填充，至少保证 `struct _heap_info`、`struct malloc_state` 等初始化依赖在进入 `maze1()` 前可用。

### TC-009: `--tar` 单入口分析能力回归
- **结果**: FAIL
- **预期行为**: `--tar` 模式应完成解包、预处理、分析与结果输出。
- **实际行为**: `--tar` 模式可以进入 `prepare.Run()`，但在分析阶段因 ptmalloc 初始化 panic 而中止。
- **复现步骤**:
  1. 执行 `python3 testdata/run_test.py python/20260129-complex-types-311`
  2. 观察 `maze --tar ... --text --json-output --rmlog --limit 500` 的输出
  3. 查看 panic 栈顶为 `maze/mallocer/ptmalloc.initPtmallloc()`
- **可能原因**: `--tar` 入口接线存在，但 prepare 到分析引擎之间关于 ptmalloc 结构信息的契约未满足。
- **建议修复**: developer 应核对 `prepare` 填充的 `common.S.Sizeof` / `common.S.Structs` 是否覆盖 ptmalloc 所有必需字段，并补充回归测试防止再次遗漏。

### TC-010: 模式兼容与遗留依赖清理检查
- **结果**: FAIL
- **预期行为**: `postmanprofile` 模式应能完成 prepare 与后续分析，输出文本和 JSON 结果。
- **实际行为**: `postmanprofile` 模式进入 `prepare.Run()` 后同样在 `initPtmalloc sizeof _heap_info` 处 panic。
- **复现步骤**:
  1. 执行 `./maze --mode postmanprofile --project coredump --user default --profileId 2636550 --pid 2636550 --text --json-output --rmlog --limit 50`
  2. 观察 panic 栈
- **可能原因**: 问题不是单一 `--tar` 包装层导致，而是 prepare 完成后进入统一分析主链路时，ptmalloc 所需结构元数据整体缺失或不完整。
- **建议修复**: developer 应优先修复 prepare 与 ptmalloc 初始化之间的数据契约问题，再重新回归 `standalone` / `--tar` / `postmanprofile` 三种模式。
