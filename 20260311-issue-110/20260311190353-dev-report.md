## Summary

Fixed three QA-identified issues blocking GDB interaction in the prepare package:

1. **GDB Server process exits immediately** — The GDB process was started without stdin, causing it to exit after executing `-ex` commands. Fixed by providing a stdin pipe (`cmd.StdinPipe()`) to keep GDB in interactive mode, allowing `gdb.post_event()` based socket server to function.

2. **tar mode readelf path** — In tar mode, `setupLanguage()` used the original file paths from maps (e.g., `/home/.../python3.11`) for `readelf`, but files were renamed by MD5 hash. Fixed by loading `{pid}.md5` JSON file to create file path mappings, and using `Realpath()` in `setupLanguage()`.

3. **Cpp.merge() simplified implementation** — The merge function was a no-op stub. Implemented the full type index remapping logic ported from the original Python `cpp_analyzer.py`, including vtable address offset, symbol address offset, type struct field remapping, and deduplication.

4. **cmd_struct_malloc_state parsing** — The `cmd_struct_malloc_state` GDB command output was executed but not parsed, causing `struct malloc_state` data to be missing. Fixed by parsing `cmd_` prefixed command output the same way as `offsets-of` output.

## Changes

- `prepare/gdb_server.go`: Added `stdinPipe` and `logFile` fields to `GDBServer`; `Start()` now creates stdin pipe to keep GDB alive; `Stop()` closes pipe gracefully; increased timeout to 10s; fixed `IsRunning()` which used invalid `os.Signal(nil)`
- `prepare/process.go`: Added `loadMd5FileMap()` to load `{pid}.md5` JSON mapping for tar mode; `setupLanguage()` now uses resolved paths via `Realpath()` with fallback for missing files; added `encoding/json` import
- `prepare/gdb.go`: `Ctypes()` now parses `cmd_` prefixed command output (e.g., `cmd_struct_malloc_state`) using the same struct/sizeof parsing logic as `offsets-of`
- `prepare/cpp_analyzer.go`: Implemented full `merge()` with type index remapping, vtable/symbol address offset application, type struct field remapping, and conversion to `common.CppStruct`/`SymbolInfo` types; added helper functions for JSON data extraction
