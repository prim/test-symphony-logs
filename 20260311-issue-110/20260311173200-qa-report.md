### `mainTarMode()` 结果文件丢失问题

```go
// main.go 第 374-376 行
origDir, _ := os.Getwd()
os.Chdir(tempDir)          // 切换到临时目录
defer os.Chdir(origDir)     // 函数退出后切回

// main.go 第 352 行
defer os.RemoveAll(tempDir) // 函数退出后删除临时目录
```

`maze-result.json` 在分析过程中写入当前工作目录（即 `tempDir`），但 `defer os.RemoveAll(tempDir)` 在 `defer os.Chdir(origDir)` **之前**注册（Go defer 是 LIFO），所以执行顺序是：
1. `os.Chdir(origDir)` — 切回原始目录
2. `os.RemoveAll(tempDir)` — 删除临时目录（包含 maze-result.json）

**建议修复**: 分析完成后、函数返回前，将 `maze-result.json` 和 `maze-result.txt` 复制到 `origDir`。

### `parseMapsLine` 字段解析偏差

在 `prepare/process.go` 第 78-111 行，maps 行解析将 `fields[6]` 赋值为 `entry.File`，但标准 maps 格式中文件路径通常是 `fields[5]`（第 6 个字段，0-indexed 为 5）。当字段 `inode` 为 0 且路径为空时，`fields[6]` 可能不存在或错位。需要核实此解析逻辑与原 Python 实现是否一致。
