## Summary

修复 standalone 模式下 HttpSetup panic 的根本原因：在 #110 迁移 Python 预处理层到 Go prepare 包时，`FillCommonVars()` 遗漏了对 `S.Common.HttpUrl` 和 `S.Http` 的默认值设置，导致 `HttpSetup()` 向裸路径 `/new` 发起 POST 时因缺少协议和主机信息而 panic。

根据 human review 的指导（"看看最近的对 ./maze 文件的提交改动，分析一下为什么没有默认配置"），追溯发现旧版 Python 入口脚本中硬编码了 `host = "http://175.178.73.182:9090"`，该值在迁移过程中丢失。修复方式是在 `FillCommonVars()` 中恢复这个默认值设置，而非简单跳过 `/new` 注册。

## Changes
- `prepare/fill.go`: 在 `FillCommonVars()` 末尾添加 `S.Common.HttpUrl` 和 `S.Http` 的默认值设置，与旧版 Python 入口脚本中的 `host = "http://175.178.73.182:9090"` 保持一致
- `http.go`: 更新 `HttpSetup()` 和 `HttpLoop()` 中的注释和日志消息，明确标注为"防御性校验"（正常流程下默认值已由 prepare.FillCommonVars 设置），将日志文案从"缺少配置"改为"无效 URL"
