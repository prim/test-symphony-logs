## Summary

根据 human review 反馈（PR #130："http.go 里面防御性代码现在是必须的吗？"），移除了 http.go 中不必要的防御性代码。

**根因分析**：问题的根本原因是 `prepare/fill.go` 中 `FillCommonVars()` 遗漏了默认 `HttpUrl` 的设置（在从 Python 入口脚本迁移到 Go prepare 包时丢失）。前一轮修复已在 `fill.go` 中正确恢复了默认值 `http://175.178.73.182:9090`，因此 http.go 中大量新增的防御性校验（`resolveHttpTarget`、`hasValidHttpBaseURL`、`HttpSetup` 和 `HttpLoop` 中的检查分支）变得多余。

**本轮修复策略**：
1. 移除 http.go 中所有新增的防御性函数和分支
2. 保留一个有价值的改进：将 `HttpJsonRequest` 中的 `C(err, ...)` panic helper 替换为 `return nil, fmt.Errorf(...)`，确保 HTTP 请求失败时不会 panic（满足 AC5）
3. 根因修复（`fill.go` 默认值恢复）保持不变

## Changes

- `http.go`: 移除 `resolveHttpTarget()`、`hasValidHttpBaseURL()`、`httpRegisterNewSession()` 函数；移除 `HttpSetup()` 和 `HttpLoop()` 中的防御性校验分支；还原 `HttpSetup()` 中的 `/new` 调用为原始简洁形式；保留 `HttpJsonRequest()` 中 `C(err,...)` → `fmt.Errorf(...)` 的改进
