## Summary
Fix standalone mode HttpSetup panic when base URL is missing. When `S.Common.HttpUrl` is empty (typical in standalone/local mode), the code now gracefully skips remote `/new` session registration instead of panicking with `unsupported protocol scheme ""`.

Key changes:
- Added `resolveHttpTarget()` helper using `net/url` for base URL validation
- `HttpJsonRequest()` now returns errors instead of panicking via `C(err)`
- `HttpSetup()` checks base URL validity before attempting `/new` registration
- `HttpLoop()` enters local-only interactive mode when no remote server is configured

## Changes
- `http.go`: Added `resolveHttpTarget()`, `hasValidHttpBaseURL()`, `httpRegisterNewSession()` functions; refactored `HttpJsonRequest()` to use URL validation and return errors; refactored `HttpSetup()` to skip `/new` when base URL is invalid; refactored `HttpLoop()` to fall back to local interactive mode
