## Findings

### 代码变更范围（静态检查）
- `http.go`：新增 `resolveHttpTarget()`、`hasValidHttpBaseURL()`、`httpRegisterNewSession()` 辅助函数；重构 `HttpJsonRequest()` 将 panic helper `C(err, ...)` 替换为 error return；`HttpSetup()` 增加 base URL 校验分支；`HttpLoop()` 增加 local-only 降级路径（`select {}` 阻塞保持 Interactive goroutine 运行）
- `tests/test_issue_126_standalone_http_setup_base_url.py`：5 个 TDD 回归测试

### 静态检查结论
1. **AC1（不再 panic）**：✅ `HttpJsonRequest()` 中所有 `C(err, ...)` 已替换为 `return nil, fmt.Errorf(...)`；`HttpSetup()` 在 base URL 无效时提前 return
2. **AC3（日志说明原因）**：✅ `Debug.Printf` 和 `Result.Printf` 均输出跳过原因和当前 `S.Common.HttpUrl` 值
3. **AC4（合法 URL 使用绝对地址）**：✅ `resolveHttpTarget()` 用 `net/url.Parse` 校验 scheme/host 后拼接绝对 URL
4. **AC5（非法 URL 返回可诊断错误）**：✅ 检查空字符串、缺失 scheme、缺失 host，均返回描述性 error
5. **AC6（tar --http 回归）**：✅ `mainTarMode()` → `mainStandalone()` → `mazehttp()` → `HttpLoop()` 路径完整未被破坏

### 注意事项（非阻塞）
- `HttpPushByReqId()` 调用 `HttpJsonRequest("push", args)` 时忽略了返回的 error。在 local-only 模式下 `HttpLoop()` 被 `select {}` 阻塞，不会进入 `HttpPull` 循环，因此不会触发 `HttpPushByReqId`。行为安全但不够优雅，建议后续优化。
