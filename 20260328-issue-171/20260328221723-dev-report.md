## Summary
修复了 TODO 审计工具和回归测试对单个带日期报告文件的硬编码依赖，让稳定入口与结构化数据成为当前正式报告路径的唯一共享来源。

## Changes
- `cmd/todo_audit.py`: 去掉固定日期报告常量，改为从结构化数据/稳定入口解析当前正式报告路径，并在校验时核对稳定入口是否声明该路径。
- `dev-log/todo-audit-data.json`: 新增 `current_report` 字段，作为工具与测试共享的当前正式报告路径声明。
- `tests/test_issue_171_todo_audit_regression.py`: 改为通过 `current_report` 定位正式报告，避免测试继续绑定固定日期文件名。

## Testing
- `python3 cmd/todo_audit.py --check`
- `python3 -m unittest tests/test_issue_171_todo_audit_regression.py`
- `./maze --build`
