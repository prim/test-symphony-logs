### TC-002: CLI 能识别并构建 refs v2
- **结果**: PASS
- **实际行为**: `maze` 中 `normalize_refs_version()` 已支持 `v2`；`build_maze()` 会为 `refs=v2` 自动补齐 `GOEXPERIMENT=arenas` 并追加 `refs_v2` build tag；`./maze --refs v2 --help` 可正常工作。
- **关键指标**: `./maze --build` 通过；`./maze --build --refs v2` 通过。
- **备注**: `refs v2` 构建路径不会再无条件注入 RocksDB 构建环境。

### TC-003: hooks 命名泛化与 v1/v2 兼容行为
- **结果**: PASS
- **实际行为**: `common/refs/shared/types.go` 同时保留 `RefsDBPath` 与 `RocksDBPath`；`common/refs.go` 保持兼容字段注入，并在 `refs.CurrentVersion() == "v2"` 时额外注入 `RefsDBPath`。`common/refs/v1/refs.go` 与 `common/refs/v2/refs.go` 都优先读取 `RefsDBPath`，为空时回退到 `RocksDBPath`。
- **关键指标**: `GOEXPERIMENT=arenas go test -tags refs_v2 ./common -run TestRefsRuntimeHooksPathSelectionByVersion -v` 通过。
- **备注**: 默认 `v1` 仍保留 `txos-rocksdb-*` 稳定路径，`v2` 使用 `txos-refsdb-*`。

### TC-004: refs v2 使用 BitalosDB 且保持薄适配层

- **备注**: 满足“不要抽象 core”的人工约束，当前 `v2` 为独立实现，不再复用会污染 `v1` 语义的共享抽象层。
