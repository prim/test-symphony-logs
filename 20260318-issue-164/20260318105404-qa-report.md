## 详细结果

### TC-001: refs v2 结构与版本选择落位
- **结果**: PASS
- **实际行为**: `common/refs/v2/refs.go`、`common/refs/select_v2.go` 存在；`select_v2.go` 仅在 `refs_v2 && goexperiment.arenas && !refs_v0 && !refs_v1` 下绑定 `v2`；`select_default.go` 仍默认绑定 `v1`。
- **关键指标**: `select_conflict.go` 已覆盖 `refs_v0 && refs_v1`、`refs_v0 && refs_v2`、`refs_v1 && refs_v2` 三类冲突；`select_v2_noarenas.go` 存在编译期兜底。
- **备注**: 符合“新增 v2、默认仍为 v1、任意两个 refs tag 同开即失败”的验收点。

### TC-002: CLI 能识别并构建 refs v2
- **结果**: PASS
- **实际行为**: `maze` 的 `normalize_refs_version()` 已支持 `v2`；`build_maze()` 会在 `refs=v2` 时追加 `refs_v2`，并自动补齐 `GOEXPERIMENT=arenas`；`./maze --build` 与 `./maze --build --refs v2` 均成功。
- **关键指标**: 构建命令返回成功；`qa/issue164` 静态测试通过。
- **备注**: 当前脚本中同时保留了 `refs_requires_rocksdb()` 与 `build_requires_rocksdb()` 语义拆分，满足最新修复目标。

### TC-003: hooks 命名泛化与 v1/v2 兼容行为
- **结果**: PASS
- **实际行为**: `common/refs/shared/types.go` 同时保留 `RefsDBPath` 与 `RocksDBPath`；`common/refs.go` 在默认/`v1` 场景仍注入 `RocksDBPath`，仅在 `refs.CurrentVersion() == "v2"` 时额外注入 `RefsDBPath`；`v1`/`v2` 内部都优先读取 `RefsDBPath`，为空再回退 `RocksDBPath`。
- **关键指标**: `GOEXPERIMENT=arenas go test -tags refs_v2 ./common -run TestRefsRuntimeHooksPathSelectionByVersion -v` 通过。
- **备注**: 默认 `v1` 路径保持 `txos-rocksdb-*`，`v2` 使用 `txos-refsdb-*`，符合“中性命名同时支持 v1 v2”的人工要求。

### TC-004: refs v2 使用 BitalosDB 且保留薄适配层
- **结果**: PASS
- **实际行为**: `common/refs/v2/refs.go` 依赖 `github.com/zuoyebang/bitalosdb`，未发现 `grocksdb` 依赖；`openDB`、`writeIndexBatch`、`writeReverseBatch`、`rangeScan` 等薄适配函数存在；对外合约 `GetReferents`、`GetReferrers`、`Save`、`Check` 均保留。
- **关键指标**: `qa/issue164/refs_v2_bitalosdb_test.go` 通过；源码静态检查确认 `rangeScan()` 会检查 `iter.Error()` 并统一关闭 iterator。
- **备注**: 满足“不抽象 core、允许 v1/v2 重复代码”的人工 review 约束。

### TC-005: arenas 前提不满足时前移为编译期失败
